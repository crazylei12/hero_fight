# 第一阶段主动大招数值设计基线

最后更新：2026-04-21

## 文档用途

这份文档用于统一第一阶段中“主动释放型、一局一次大招”的数值设计口径。

它主要解决以下问题：
- 一局一次的大招到底应该强到什么程度
- 瞬时伤害、持续伤害、瞬时治疗、持续治疗、群体护盾应该分别对齐什么结果
- 带控制、减速、保护或其他特殊效果时，原始伤害或治疗应扣掉多少预算
- 为什么不同职业的大招倍率可能看起来差很多，但仍然是合理的

这份文档优先服务：
- 规划与文档统一
- 后续英雄规格编写
- Unity 内实际调数时的目标锚点

它不是要求今天就把所有 `.asset` 数值一次改完，而是先把“调数时要朝哪里收口”写成正式规则。

## 适用范围

- 只适用于 `主动释放型` 且 `一局一次` 的大招
- 当前战斗规则默认仍然是：
  - `5v5`
  - `60 秒`
  - 死亡后 `5 秒` 复活
  - 统一伤害体系
- 如果某个英雄未来的大招改成被动触发、可多次使用或极端特例机制，应在英雄文档中显式说明偏离原因

## 当前生存锚点

第一阶段后续讨论大招强度时，默认先按下面这组“参考承伤档位”估算，不要每次都重新凭感觉描述“脆皮”和“坦克”。

### 标准脆皮锚点

- 参考对象：`Firemage` / `Frostmage` 一类后排法师
- 参考属性：
  - `300` 生命
  - `10` 防御
- 对统一伤害公式而言，对应有效承伤约为 `330`

### 极脆脆皮锚点

- 参考对象：`Rifleman` 这类低防低血后排
- 参考属性：
  - `280` 生命
  - `6` 防御
- 对统一伤害公式而言，对应有效承伤约为 `297`

### 中坚锚点

- 参考对象：`Skybreaker` / `Monk` 一类能持续站场的近战
- 参考属性：
  - `420` 生命
  - `24` 防御
- 对统一伤害公式而言，对应有效承伤约为 `521`

### 坦克锚点

- 参考对象：`Ironwall` 一类主坦克
- 参考属性：
  - `560` 生命
  - `40` 防御
- 对统一伤害公式而言，对应有效承伤约为 `784`

补充说明：
- 后续如果某个新英雄明显超出这些档位，应先在英雄文档里声明它属于新的承伤档，再决定是否重调大招基线
- `伤害大招` 优先按“结算后的实际掉血结果”衡量
- `治疗与护盾大招` 优先按“实际抬起或保护的生命值结果”衡量，不要直接拿 raw multiplier 横比

## 核心原则

- 一局只能主动释放一次的大招，强度目标必须明显高于小技能，不应只看成“CD 更长”
- 大招调数优先看 `最终战局摆动`，不是只看倍率写起来大不大
- `持续伤害 / 持续治疗` 因为实战常常打不满、吃不满，满额吃满时的总量应高于瞬时技能
- `范围覆盖越大`、`控制或保护越强`，直接伤害或直接治疗的原始数值就越要往下扣
- 不同职业的 `attackPower` 基线差异很大，所以：
  - 输出位的大招倍率不必和辅助完全同级
  - 辅助的大招倍率看起来更大，可能反而是正常的

## 按形态划分的结果锚点

下面所有百分比，都优先指“对目标当前满血状态造成或恢复的实际生命值比例结果”。

### 1. 瞬时单体或窄命中纯伤害大招

适用例子：
- 狙击
- 神枪手点杀
- 单体斩杀
- 命中人数通常不高的窄路径穿刺

结果锚点：
- 对标准脆皮：应达到 `100% ~ 120%`
- 对中坚：应达到 `70% ~ 85%`
- 对坦克：应达到 `45% ~ 60%`

解释：
- 这类技能的正式目标就是“满血秒掉脆皮”
- 坦克吃完后仍应活着，但必须进入明显不健康状态，不能像吃了个强化小技能

### 2. 瞬时范围纯伤害大招

适用例子：
- 大范围爆炸
- 范围炮击
- 群伤手榴弹

结果锚点：
- 对标准脆皮：满额命中时应达到 `100% ~ 120%`
- 对中坚：应达到 `70% ~ 85%`
- 对坦克：应达到 `50% ~ 65%`



### 3. 持续 / 多段纯伤害大招

适用例子：
- 持续火海
- 雷暴区域
- 多段连斩
- 可打断但完整落完后总伤害很高的连续输出大招

结果锚点：
- 对标准脆皮：满额吃满应达到 `170% ~ 220%`
- 对中坚：满额吃满应达到 `120% ~ 150%`
- 对坦克：满额吃满应达到 `100% ~ 115%`

解释：
- 这类技能实战很难稳定打满，所以打满时就应该非常痛
- 这也是火法一类“持续伤害大招”应明显高于瞬时爆发档位的原因
- 如果一个持续伤害大招打满后连坦克都杀不掉，它很容易在第一阶段的一局一次规则下显得存在感不足

### 4. 持续 / 多段伤害 + 明显控制或压制效果的大招

适用例子：
- 持续范围伤害 + 高额减速
- 持续范围伤害 + 稳定推拉
- 持续范围伤害 + 高频控制覆盖

结果锚点：
- 对标准脆皮：满额吃满应达到 `130% ~ 180%`
- 对中坚：满额吃满应达到 `90% ~ 120%`
- 对坦克：满额吃满应达到 `70% ~ 90%`

解释：
- 这类技能的价值已经不只在“掉血”
- 因为它额外贡献了大范围控场、打断推进、压缩走位等收益，所以直接伤害应低于纯伤害持续大招
- `Blizzard` 一类技能默认应落在这个档位，而不是继续对齐火法的纯伤害档

### 5. 瞬时治疗大招

适用例子：
- 瞬时大额抬血
- 爆发救人
- 单体或小范围直接治疗

结果锚点：
- 对标准脆皮：应恢复 `70% ~ 95%`
- 对中坚：应恢复 `55% ~ 75%`
- 对坦克：应恢复 `35% ~ 50%`

解释：
- 瞬时治疗大招的目标是把关键队友从击杀线直接拉回安全线
- 它不一定要把坦克从残血瞬间抬满，但必须让坦克从“下一轮就死”回到“还能继续扛”

### 6. 持续治疗大招

适用例子：
- 持续治疗区域
- 多段治疗结界
- 团战续航场

结果锚点：
- 对标准脆皮：满额吃满应恢复 `140% ~ 190%`
- 对中坚：满额吃满应恢复 `100% ~ 135%`
- 对坦克：满额吃满应恢复 `90% ~ 105%`

解释：
- 逻辑和持续伤害大招一致：因为很难稳定吃满，所以吃满时总量必须高
- 如果一个持续治疗大招吃满后都顶不住一次敌方伤害大招，那它很难承担“一局一次”的强技能定位
- 但对多人持续范围治疗来说，默认应理解为“主要价值集中在 `1~2` 名核心友军身上”，不要求 `5` 人同时完整吃满时仍保持单体极限治疗强度

### 7. 群体护盾 / 减伤 / 保护型大招

适用例子：
- 群体护盾
- 范围减伤
- 保护结界
- 团队承伤窗口

结果锚点：
- 当稳定覆盖 `3` 名及以上友军时，单个目标的原始保护量可以低于瞬时单体治疗档
- 推荐每个目标先按以下区间估算：
  - 标准脆皮：`50% ~ 70%`
  - 中坚：`35% ~ 55%`
  - 坦克：`25% ~ 45%`

解释：
- 这类大招的强度主要来自“同时抬高多名友军的有效生命”
- 不应再强求它对单个目标也达到瞬时单体治疗大招的极限抬血强度
- 如果还额外附带：
  - 强减伤
  - 无敌
  - 不可选中
  - 分担伤害
  - 群体硬控反制
  则单个目标的 raw 护盾值还要继续下调

## 覆盖与特殊效果折损规则

设计新大招时，建议先按“纯伤害 / 纯治疗”确定结果档位，再按下面两类折损往下扣。

### 覆盖折损

- 单体、窄路径、稳定只打 `1` 人：不折损
- 小范围、常见命中 `2 ~ 3` 人：直接数值下调约 `10% ~ 20%`
- 大范围、常见命中 `4+` 人或全队：直接数值下调约 `25% ~ 40%`

### 特殊效果折损

- 轻度附加效果：
  - 一般减速
  - 短位移
  - 小幅 Debuff
  - 轻量辅助状态
  建议下调约 `10% ~ 20%`
- 明显强效附加效果：
  - 高额减速且覆盖稳定
  - 眩晕 / 击飞 / 嘲讽
  - 显著护盾 / 减伤
  - 分担伤害
  - 短时无敌 / 不可选中
  建议下调约 `25% ~ 40%`

补充规则：
- 覆盖折损和特殊效果折损可以叠加
- 一旦同时拥有“大覆盖 + 强控制 / 强保护”，就不应继续追求“还要满血秒脆皮”的纯伤害目标

## 可实现性补偿规则

如果某个大招在玩法上本来就难打满、难命中或容易被打断，可以适当把总量往上补。

允许补偿的常见情况：
- 固定区域且敌人容易走出
- 有明显延迟，敌人可能离开落点
- 连续动作序列，期间可能被硬控打断
- 需要较严格的站位条件才吃到全部收益

建议补偿幅度：
- 总预算上调约 `10% ~ 20%`

约束：
- 如果技能已经接近“稳定必中”或“很难反制”，就不要再同时拿满这层补偿

## 换算提醒

当前统一伤害公式：

```text
finalDamage = attackPower * powerMultiplier * critMultiplier * (100 / (100 + defense))
```

在不考虑暴击时：

```text
damageMultiplier ≈ 目标实际掉血 / (施法者攻击力 * (100 / (100 + 目标防御)))
```

持续技能如果预计完整命中 `N` 次：

```text
singleTickMultiplier ≈ totalMultiplier / N
```

当前统一治疗公式：

```text
finalHeal = attackPower * powerMultiplier
```

因此：
- 治疗大招优先按“要恢复多少实际生命值”反推倍率
- 辅助的 `attackPower` 明显低于输出位，所以治疗倍率看起来更大是正常现象

## 当前样例英雄应收口到的方向

### Firemage / Meteor Fall

- 应按“持续 / 多段纯伤害大招”设计
- 单个标准脆皮吃满应稳定击杀
- 满血坦克吃满应允许被单独击杀
- 因为实战通常打不满，所以总量应显著高于瞬时爆发档

### Frostmage / Blizzard

- 应按“持续 / 多段伤害 + 明显控制”设计
- 单个标准脆皮吃满应稳定击杀
- 中坚吃满应重伤
- 满血坦克吃满应被压进危险线，但不以单独击杀为默认目标

### Longshot / Rapid Barrage

- 应按“可打断的连续输出终结大招”设计
- 如果 `20` 发完整集中到一个满血脆皮，应稳定击杀
- 如果完整集中到一个满血坦克，应明显压低其血线，但通常应更依赖前置压血或团队跟进完成收口

### Rifleman / Frag Grenade

- 应按“瞬时范围纯伤害大招”设计
- 满额命中满血脆皮时，应接近或直接击杀
- 中坚应被重伤
- 坦克应存活但不健康

### Sunpriest / Sun Blessing

- 应按“持续治疗大招”设计
- 若主要收益集中在 `1~2` 名核心友军身上，脆皮吃满应从危险线回到安全线
- 中坚或坦克吃满应从危险线回到可继续接战的健康线

### Monk / Guardian Mantra

- 应按“群体护盾 / 保护型大招”设计
- 不追求单个目标获得等同于单体爆发治疗大招的 raw 数值
- 但当 `3` 名以上友军稳定吃到收益时，团队总等效承伤必须足以顶住一次敌方强爆发窗口

## 对后续 AI 的约束

后续 AI 在调整第一阶段大招时，必须遵守：
- 不要再按“小技能倍率稍微放大一点”去理解一局一次大招
- 不要只盯 raw multiplier，必须换算成结算后的实际掉血 / 抬血结果
- 不要让带明显控制、减伤、护盾或其他特殊效果的大招，还同时保留纯伤害大招的满额输出
- 不要因为辅助的治疗倍率数字看起来大，就误判为超标；先结合辅助自身攻击力再判断
- 如果某个英雄需要偏离本文件的通用档位，必须在英雄文档中写明偏离原因
