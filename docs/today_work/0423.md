### 本轮实际接入的策略与参数
- 通用节奏参数：
- 开局统一大招锁定：`6.0s`
- 大招判定间隔：`0.75s`
- 每轮判定随机抖动：`0.5s`
- 基础释放概率：`0.30`
- 主条件超额单位 bonus：每超出 `1` 个单位，`+0.15`
- fallback 第一阶段 bonus：`+0.20`
- fallback 第二阶段 bonus：`+0.50`
- secondary priority bonus：`+0.25`

- 时机档 `Early`：
- 个人团队级开大窗口随机值：`6.0s ~ 10.0s`
- 说明：实际仍受统一 `6.0s` 开局锁定限制，不会早于开场 `6s` 前尝试

- 时机档 `Standard`：
- 个人团队级开大窗口随机值：`10.0s ~ 16.0s`
- 说明：当前默认档位；更接近“中段接敌后再逐步开大”

- 时机档 `Late`：
- 个人团队级开大窗口随机值：`16.0s ~ 24.0s`
- 说明：在 `60s` 小局里已经属于偏晚档，会明显推迟大招进入团队级许可窗口

- 协同档 `Together`：
- 队友刚开大后的提前跟大窗口：`1.25s`
- 概率乘数：`1.35`
- 额外行为：如果本英雄自己的团队级时机窗还没到，但队友刚在 `1.25s` 内开过大，则允许提前进入“可尝试开大”的状态

- 协同档 `Standard`：
- 中等协同修正窗口：`2.5s`
- 概率乘数：`0.65`
- 额外行为：没有硬锁，只是中等强度地下调队友开大后的跟大概率

- 协同档 `Separate`：
- 硬延后窗口：`3.0s`
- 额外概率抑制窗口：`5.0s`
- 概率乘数：`0.25`
- 额外行为：如果本英雄已经进入自己的团队级时机窗，但队友在最近 `3.0s` 内刚开过大，则本轮直接不尝试；即使过了这段硬延后，在最近 `5.0s` 内仍会吃到 `0.25` 的强概率抑制

- 默认值：
- `BattleTeamLoadout` 默认：`ultimateTimingStrategy = Standard`
- `BattleTeamLoadout` 默认：`ultimateComboStrategy = Standard`
- `Stage01SampleContentBuilder` 生成的样例输入也已同步设置为蓝红双方默认 `Standard / Standard`

### 参数适配关注点
- `Early 6~10s`：因为统一锁定本身就是 `6s`，这个档位的前半段实际上会集中在“刚解锁就能尝试”的区间，可能偏激进。

- `Late 16~24s`：在 `60s` 局里占比不低，适合保守阵容，但可能会让一局一次大招的存在感延后太多。
- `Together 1.25s + 1.35x`：具备明确“跟大”机制，但倍率提升不算特别高，更多是“允许跟上”，未必能形成非常强的同时爆发。
- `Standard 2.5s + 0.65x`：如果项目希望“适中”更接近真正中立，这个抑制可能略重。
- `Separate 3s 硬锁 + 5s 0.25x`：当前是明显最强的错峰档，后续最值得先看这一档是否过头。
