# Yasuo

## 基础信息

- 英文名：Yasuo
- 中文名：亚索
- 英雄 ID：`warrior_007_yasuo`
- 英雄分类：战士
- 标签：近战、协同追击、护盾、击飞联动、条件触发型大招
- 核心定位：极度依赖队友击飞的近战协同战士，自身不提供任何击飞、眩晕或强制位移控制；当队友把敌人打到空中时，亚索会立刻跟进并对每一个被击飞的敌人造成追加伤害
- 默认 AI 模板：`warrior_default`
- 当前定位补充：作为 `warrior_006_chainbreaker` 之后的第七个战士扩展原型，不替代 `Skybreaker` 的主动开团击飞、`Spellblade` 的阵型压缩、`TrollWarlord` 的单挑叠层或 `Chainbreaker` 的反控制模板；亚索主要验证“友方控制事件监听 + 多目标触发伤害 + 条件触发型大招位”这条强组合依赖方向

## 当前数值倾向

- 生命值：中等，不能像主坦一样长期吃集火
- 防御力：中等偏低，生存主要依赖小技能护盾和正确跟进时机
- 攻击力：中高，确保击飞联动触发时有足够爆点
- 攻速：中等，常态输出不能过低，但不应压过大招联动收益
- 移速：标准战士偏高，方便跟随前排接敌
- 攻击距离：标准近战，不把他做成远程消耗角色
- 暴击收益：中等，允许有剑客爆发感，但核心强度仍来自击飞联动而不是随机暴击

## 建议实现基线

- 最大生命：`410`
- 攻击力：`44`
- 防御力：`18`
- 攻速：`1.0`，约等于 `1.00 秒 / 次`
- 移速：`4.55`
- 暴击率：`12%`
- 暴击伤害：`1.65`
- 攻击距离：`1.8`

## 普攻类型

- 类型：近战瞬时命中普攻
- 释放目标：当前索敌目标；战士默认优先接敌并持续攻击最近合法敌人，当前目标失效后重新追击最近敌人
- 效果：造成基于攻击力 `1.0` 倍率的单体伤害
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`1.8`
  - 普攻间隔：`1.00 秒`
- 当前关键约束：
  - 普攻不附带击飞、眩晕、减速、破甲、吸血或额外叠层
  - 亚索自身不能通过普攻制造大招触发条件

## 小技能：Windstep / 踏风突进

- 技能类型：随机敌方目标突进 + 自身护盾
- 释放目标：施法范围内随机 `1` 名敌方英雄；随机结果必须在施法开始时锁定，并通过战斗随机源和日志记录，不允许表现层重新随机
- 效果：
  - 施放瞬间为自己附加一个短持续护盾
  - 随后向锁定目标冲刺，并落到目标附近的合法近战位置
  - 冲刺本身不造成伤害
  - 冲刺本身不附加击飞、眩晕、减速、嘲讽、强制位移或集火标记
  - 若目标在冲刺途中死亡、进入复活等待、变为不可选中或阵营失效，则亚索落到冲刺路径上最后一个合法位置，不重新寻找新目标
  - 该技能的核心作用是让亚索主动进入可以普攻和等待队友击飞的交战位置，而不是提供独立开团能力
- 检测条件：
  - 小技能 CD 转好
  - 自身不处于 `眩晕`、`击飞` 或其他阻止主动施法的状态
  - `4.5` 范围内至少存在 `1` 名合法敌方英雄
  - 若自身已有护盾仍可释放，但新护盾按统一护盾叠加 / 覆盖规则处理，不私自维护第二套护盾池
- 当前关键参数：
  - 施法距离：`4.5`
  - 冲刺过程时长：`0.22 秒`
  - 落点距离目标：`1.25`
  - 护盾值：`35`
  - 护盾持续时间：`2 秒`
  - 冷却时间：`3 秒`

## 大招：Last Breath / 狂风绝息

- 技能类型：条件触发型大招 / 友方击飞监听 / 多目标追加伤害 / 短距离闪烁落位
- 释放目标：被己方其他英雄成功附加 `KnockUp` 的敌方英雄集合
- 偏离主动大招的原因：
  - 该机制承载亚索最高优先级的战局身份：只有队友先制造击飞，他才获得爆发跟进
  - 如果改成主动释放型大招，就会弱化“依赖队友特定控制”的核心限制
  - 因此它占用正式大招位，不再额外拥有一局一次主动大招
- 触发条件：
  - 亚索处于存活状态，且不在复活等待中
  - 敌方英雄被亚索的队友成功附加 `KnockUp`
  - 触发来源必须是己方其他英雄；亚索自身的普攻、小技能和大招都不能制造触发条件
  - 目标必须是亚索的合法敌方英雄，且不是死亡、复活等待、不可选中或阵营失效状态
  - 同一名敌人、同一次 `KnockUp` 施加事件只触发 `1` 次，不因击飞持续时间逐帧重复触发
- 多目标触发规则：
  - 若同一帧或同一批技能结算中有多个敌方英雄被击飞，亚索应对这些敌人全部造成伤害
  - 同一批触发按一个触发批次处理，先收集全部合法被击飞敌人，再统一结算伤害和落点
  - 每名被击飞敌人各自受到一次基于攻击力 `1.45` 倍率的伤害
  - 伤害不因本批次目标数量增加而额外提升，也不因目标数量增加而分摊降低
  - 若后续又发生新的合法 `KnockUp` 施加事件，可以再次触发；该大招不是一局一次主动资源
- 闪烁与落地规则：
  - 触发时亚索表现为闪烁到被击飞目标上方并发动斩击
  - 运行时最终位置应从本批次被击飞敌人中随机选 `1` 名作为落地锚点
  - 亚索最终落到该锚点附近的合法近战位置
  - 如果随机落地锚点在落地前死亡、进入复活等待、不可选中或阵营失效，则从本批次仍合法的其他被击飞敌人中重新随机；若本批次已无合法锚点，则回退到触发前自身位置附近的最近合法点
  - 闪烁和上方斩击只表达跟进与落位，不提供无敌、不可选中、免伤、免控或死亡防止
- 当前关键参数：
  - 触发来源：己方其他英雄成功施加的 `KnockUp`
  - 伤害倍率：每名被击飞敌人基于攻击力 `1.45`
  - 批次收集窗口：同一帧 / 同一技能结算批次
  - 落点距离锚点：`1.25`
  - 触发次数：不设一局一次限制，但同一次 `KnockUp` 对同一目标只触发一次
- 当前关键约束：
  - 大招不创建 `KnockUp`
  - 大招不延长、刷新或复制队友造成的 `KnockUp`
  - 大招不打断目标当前已经处于的击飞，只额外结算伤害与亚索落位
  - 大招不应绕过统一伤害结算、状态合法性、事件和日志系统

## 落地约束

- 亚索的大招监听应复用共享状态事件或等价的 `StatusAppliedEvent` 入口，过滤 `StatusEffectType.KnockUp` 与友方来源，不要写成每个击飞英雄手动调用亚索的私有接口
- `Last Breath` 的触发批次必须支持一次收集多个被击飞敌人；不要只处理第一个被击飞目标后提前返回
- 多目标伤害应继续走统一伤害结算与事件日志，每个目标都要能记录实际伤害、来源技能和触发的击飞来源
- 大招落地随机必须使用战斗统一随机源，保证回放、日志和测试能复现，不要使用表现层或 Unity 全局随机临时决定
- 大招的“闪烁到上方”优先作为短动作表现与最终落位处理，不应新增完整空中物理系统
- 如果正式实现需要“正在执行跟进斩击”这类短动作锁定，应收敛为共享动作序列 / 短动作状态，不要引入亚索专属行为门禁
- 小技能护盾必须复用正式护盾语义，不要直接改写生命值或临时防御公式
- 小技能随机目标必须在施法开始时锁定，并在后续冲刺、事件、日志和表现中使用同一个目标引用或目标快照
- 事件和日志至少要能明确记录：小技能随机目标、护盾获得、冲刺开始与落点、击飞触发来源、大招命中的目标列表、每个目标实际伤害、最终随机落地锚点

## 设计边界

- 亚索的核心职责是击飞联动爆发，不承担主坦式长期承伤职责
- 亚索自身不应拥有任何击飞、眩晕、群体控制或稳定打断能力
- 不应让小技能同时承担突进、伤害、控制和高额护盾；当前小技能只负责进场与短保护
- 不应让大招在敌方只是被击退、拉拽、减速、嘲讽或眩晕时触发；触发条件必须是正式 `KnockUp`
- 不应把大招扩成“友方任意控制都能触发”的泛控制追击，否则会失去队友击飞依赖这一核心限制
- 不应让大招提供无敌、不可选中、免控、死亡防止或清除负面状态
- 不应因为大招是被动触发就额外补一个主动大招；它已经占用亚索的正式大招位
- 第一阶段下，亚索优先验证“近战普攻 + 随机突进护盾 + 友方击飞事件触发多目标跟进”的强协同战士模板
