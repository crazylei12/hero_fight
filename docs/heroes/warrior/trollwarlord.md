# TrollWarlord

## 基础信息

- 英文名：TrollWarlord
- 中文名：巨魔战将
- 英雄 ID：`warrior_005_trollwarlord`
- 英雄分类：战士
- 标签：近战、持续输出、单挑、攻速叠层、前排压制
- 核心定位：近战单挑压制战士，依靠持续攻击同一目标不断抬高攻速，专门惩罚站在前排、短时间内无法摆脱他的坦克或战士；当目标频繁切换、被控制打断或战局进入多人混战时，输出上限会明显下降
- 默认 AI 模板：`warrior_default`
- 当前定位补充：作为 `warrior_004_spellblade` 之后的第五个战士扩展原型，不替代 `Skybreaker` 的开团控制、`Bladesman` 的破甲穿切、`Berserker` 的残血续战或 `Spellblade` 的阵型压缩模板，主要验证“同目标普攻攻速叠层 + 单体强化 + 一局一次死亡防止窗口”这条偏单挑的战士方向

## 当前数值倾向

- 生命值：中等偏高，需要能在前排单挑中撑到攻速叠满
- 防御力：中等偏高，允许他贴住前排持续换血，但不做主坦式长期承伤
- 攻击力：中等，基础单次伤害不能过高，主要输出来自攻速叠层与小技能窗口
- 攻速：基础偏低，避免开局就拥有过强站桩 DPS
- 移速：标准战士偏低，不靠高速切后排，而是黏住已经接上的前排目标
- 攻击距离：标准近战，靠小技能临时拉长攻击距离来稳定追砍
- 暴击收益：偏低，输出体验应来自可预期的攻速成长，不依赖暴击爆点

## 建议实现基线

- 最大生命：`435`
- 攻击力：`39`
- 防御力：`24`
- 攻速：`0.833333`，约等于 `1.20 秒 / 次`
- 移速：`4.25`
- 暴击率：`8%`
- 暴击伤害：`1.55`
- 攻击距离：`2.0`

## 普攻类型

- 类型：近战瞬时命中普攻 + 同目标攻速叠层
- 释放目标：当前索敌目标；巨魔战将应优先接住最近前排目标并保持攻击同一目标，只有当前目标死亡、失效、变为不可选中、被强制目标效果覆盖或超过追击边界时才重新索敌
- 效果：
  - 普攻造成基于攻击力 `1.0` 倍率的单体伤害
  - 每次成功普攻命中同一个目标后，获得 `1` 层 `Fervor` 攻速叠层
  - 每层 `Fervor` 提供 `+16%` 攻速
  - 最多叠加 `6` 层，满层为 `+96%` 攻速
  - 攻击同一目标时持续累积层数；切换目标会立即清空层数，并需要从新目标重新开始叠
  - 目标只是暂时离开攻击距离时不立刻清层，只要巨魔战将仍保持该目标且没有转火，层数保留
  - 目标死亡、进入复活等待、变为不可选中、阵营失效或被嘲讽等强制目标效果改写当前目标时，视为目标切换并清空层数
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`2.0`
  - 基础普攻间隔：`1.20 秒`
  - 每层攻速提升：`+16%`
  - 最大层数：`6`
  - 满层攻速提升：`+96%`

## 小技能：Warlord Reach

- 技能类型：自身增益 Buff / 攻击力提升 / 攻击距离提升
- 释放目标：自己
- 效果：
  - 进入 `4 秒` 的长柄战斧姿态
  - 攻击力提升 `+25%`
  - 攻击距离提升 `+45%`
  - 该技能的核心作用是让巨魔战将更稳定地维持当前单挑目标，不因为前排轻微位移或最小间距收束而频繁丢失攻击节奏
  - 小技能不提供范围伤害、硬控、位移、护盾或吸血
- 检测条件：
  - 小技能 CD 转好
  - 当前存在合法敌方目标
  - 当前目标距离不超过 `2.8`
  - 若当前已有 `Fervor` 层数，释放优先级提高；避免在完全未接敌时空放
- 当前关键参数：
  - 持续时间：`4 秒`
  - 攻击力提升：`+25%`
  - 攻击距离提升：`+45%`
  - 冷却时间：`7.5 秒`

## 大招：Deathless Frenzy

- 技能类型：自身增益 Buff / 死亡防止 / 攻速叠层覆写
- 释放目标：自己
- 效果：
  - 进入 `4.5 秒` 的不灭狂怒状态
  - 持续期间，来自普通伤害、技能伤害和持续伤害的致死结算不会让巨魔战将生命值降到 `0`
  - 若受到非斩杀类致死伤害，生命值最低被锁定为 `1`
  - 这不是 `无敌`：伤害仍然正常结算、进入日志、触发贡献统计，也仍然会消耗护盾
  - 这不是 `不可选中`：敌人仍可以继续普攻、技能命中或控制巨魔战将
  - 这不是免控或霸体：眩晕、击飞、嘲讽等状态仍应照常生效；被控制时即使攻速维持满层，也不能绕过行为门禁继续攻击
  - 大招持续期间，巨魔战将的普攻攻速按 `Fervor` 满层计算
  - 大招持续期间如果发生目标切换，输出攻速仍暂时维持满层；但底层真实 `Fervor` 层数仍按当前目标重新维护
  - 大招结束后，攻速回到真实 `Fervor` 层数对应的数值；如果期间没有稳定攻击同一目标，则可能直接回落到低层甚至 `0` 层
  - 若未来加入 `斩杀 / Execute / 直接击杀` 机制，`Deathless Frenzy` 不免疫这类效果；满足斩杀条件时仍会死亡并正常计分
- 检测条件：
  - 本局尚未释放过大招
  - 当前存在合法敌方目标
  - 优先在自身生命值比例 `<= 35%` 且当前目标距离不超过 `2.8` 时释放
  - 若开战 `45 秒` 后仍未释放，只要自身生命值比例 `<= 50%` 且当前存在可攻击目标，即可释放
- 当前关键参数：
  - 持续时间：`4.5 秒`
  - 生命锁定下限：`1`
  - 攻速覆写：按 `Fervor` 满层，即 `+96%` 攻速
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`
  - 大招不免疫未来的斩杀类效果

## 落地约束

- `Fervor` 应优先作为 `BasicAttackData` 驱动的共享“同目标持续普攻叠层”能力，不要写成 `TrollWarlordOnly` 的私有计数器
- `Fervor` 的目标绑定必须使用运行时目标标识，不应只按英雄配置 ID 判断；同一英雄在双方或同阵容重复出现时也要能正确区分目标
- `Fervor` 提供的攻速收益应继续走统一属性修正或等价的正式攻速覆写入口，不要直接改写普攻计时器内部常量
- 巨魔战将需要一个轻量目标保持规则：当当前目标仍合法且未超过追击边界时，AI 不应因为最近敌人发生细小变化就频繁切目标
- 这条目标保持规则只服务“持续攻击同一目标”的战斗逻辑，不升级成完整仇恨系统；被嘲讽、目标不可选中、目标死亡等正式覆盖仍然必须生效
- `Warlord Reach` 的攻击力和攻击距离提升应分别复用正式 `AttackPowerModifier` 与 `AttackRangeModifier` 语义
- `Deathless Frenzy` 需要新增或复用共享的“死亡防止 / 生命锁定”状态语义；它与 `Invulnerable` 不同，不能实现成伤害无效
- 死亡防止期间仍应记录受到的实际伤害、伤害来源、护盾消耗、状态添加和状态结束，方便调试“为什么没死”和“斩杀为什么仍然杀死”
- 大招结束时不应自动补血；如果结束后生命值仍为 `1`，下一次普通伤害即可正常击杀

## 设计边界

- 巨魔战将的核心职责是单体持续压制，不承担主坦克式长期承伤职责
- 不应把巨魔战将做成高机动切后排刺客；他应该主要惩罚站在前排的坦克、战士或被迫与他缠斗的目标
- 不应给 `Fervor` 增加多目标保留层数、全局攻速成长或跨死亡保留层数，否则会削弱“切换目标清零”的核心反制点
- 不应把 `Warlord Reach` 扩成范围顺劈、位移追击或硬控技能；它只负责提高单体贴身输出稳定性
- 不应把 `Deathless Frenzy` 扩成无敌、不可选中、免控、净化或回血大招；它只是短时间不被普通致死伤害击杀，并把攻速维持到最大值
- 不应让死亡防止免疫未来的斩杀类效果；斩杀机制如果后续加入，应成为这类“低血不死窗口”的明确反制
- 第一阶段下，巨魔战将优先验证“近战普攻 + 同目标攻速叠层 + 临时攻击距离 / 攻击力强化 + 一局一次死亡防止”的单挑型战士模板
