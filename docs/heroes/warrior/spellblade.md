# Spellblade

## 基础信息

- 英文名：Spellblade
- 中文名：魔剑客
- 英雄 ID：`warrior_004_spellblade`
- 英雄分类：战士
- 标签：近战、路径伤害、阵型压缩、持续输出
- 核心定位：中前排压阵战士，通过对当前目标方向释放直线剑气顺带切到同线敌人，再用一局一次的留场魔剑把交战区逐步往自己脚下收紧，让己方近战和范围技能更容易打出多人收益
- 默认 AI 模板：`warrior_default`
- 当前定位补充：作为 `warrior_003_berserker` 之后的第四个战士扩展原型，不替代 `Skybreaker` 的先手开团控制模板，不替代 `Bladesman` 的破甲穿切模板，也不替代 `Berserker` 的残血续战模板，主要验证“近战站场 + 当前目标直线剑气 + 一局一次阵型压缩魔剑”这条带少量 AoE 的战士方向

## 当前数值倾向

- 生命值：中等偏高，需要能在前排站稳并把大招锚点留在交战中心
- 防御力：中等，不靠纯承伤吃完整轮集火
- 攻击力：中高，单体稳定输出要合格，但不追求像 `Bladesman` 那样的高爆发单刀
- 攻速：中等，保证普攻与小技能之间的持续压血节奏
- 移速：标准战士水平，不靠超长突进切后排
- 攻击距离：标准近战偏长，方便在前排缠斗时更稳定地把剑气对准当前目标方向
- 暴击收益：不高，主要强度不依赖暴击爆点

## 建议实现基线

- 最大生命：`425`
- 攻击力：`43`
- 防御力：`21`
- 攻速：`0.961538`，约等于 `1.04 秒 / 次`
- 移速：`4.4`
- 暴击率：`10%`
- 暴击伤害：`1.6`
- 攻击距离：`1.8`

## 普攻类型

- 类型：近战瞬时命中普攻
- 释放目标：当前索敌目标；战士默认优先接敌并黏住最近合法敌人，当前目标失效后继续追击最近敌人
- 效果：造成基于攻击力 `1.05` 倍率的单体伤害；常态普攻不额外附带控制或 Debuff，范围收益主要留给剑气和大招后的站位压缩
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`1.8`
  - 普攻间隔：`1.04 秒`

## 小技能：裂空剑气

- 技能类型：直线路径伤害
- 释放目标：默认对准当前目标释放；若当前目标不合法或超出施法距离，则按共享 `NearestEnemy` 回退到最近合法敌人；施法开始后剑气方向锁定，不在飞行中追踪目标
- 效果：
  - 向目标方向发出一道高速剑气
  - 对路径条带上的所有敌方英雄造成一次基于攻击力 `1.25` 倍率的伤害
  - 同一敌人在同一次剑气中只会受伤 `1 次`
  - 技能核心收益是让魔剑客在维持近战职责的同时，顺带切到前后排同线目标，而不是把自己变成远程炮台
- 检测条件：
  - 小技能 CD 转好
  - 施法距离 `4.8` 内存在合法敌方目标
- 当前关键参数：
  - 施法距离：`4.8`
  - 路径长度：`7.2`
  - 路径宽度：`2.2`
  - 路径伤害：基于攻击力 `1.25`
  - 冷却时间：`6.5 秒`

## 大招：缚阵魔剑

- 技能类型：部署型固定战斗代理体 / 周期脉冲拉拽 / 阵型压缩
- 释放目标：自己当前位置；魔剑客在施法点落下一把固定存在的魔剑
- 效果：
  - 在当前位置生成一把持续留场的魔剑锚点
  - 魔剑不可移动、不作为英雄单位，默认 `不可选中 + 无敌`
  - 魔剑持续期间，每 `1 秒` 以自己为圆心进行一次脉冲
  - 每次脉冲都会把大范围内的敌方英雄沿朝向魔剑的方向拉近一段距离
  - 单次拉拽采用短距离强制位移，不附带 `KnockUp`、眩晕或其他硬控
  - 若某个敌人已经非常靠近魔剑，则本次脉冲只会把它拉到魔剑周围的合法近身环，不会把多人叠进同一个点
  - 大招当前不依赖高额直接伤害；核心收益是把敌方阵型逐步压缩到一个更利于近战追砍和范围技能覆盖的位置
- 检测条件：
  - 本局尚未释放过大招
  - 当前存在合法敌方目标
  - 当前目标进入 `2.4` 范围内
  - 优先要求以自身为圆心 `8.5` 范围内至少存在 `3` 名敌方英雄
  - 若开战 `45 秒` 后仍未满足，则阈值降低为至少 `2` 名敌方英雄
- 当前关键参数：
  - 魔剑持续时间：`4 秒`
  - 脉冲间隔：`1 秒`
  - 脉冲次数：`4`
  - 拉拽半径：`8.5`
  - 单次拉拽距离：`1.8`
  - 单次强制位移时长：`0.28 秒`
  - 魔剑近身环：距魔剑 `1.4 ~ 2.4`
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 魔剑客当前定位为“第四个战士扩展原型”，不替代 `Skybreaker` 的开团控制、`Bladesman` 的破甲切线或 `Berserker` 的残血反打
- `裂空剑气` 必须优先复用统一的“锁定方向后结算一次路径伤害”能力，不要为魔剑客单独发明手动瞄准、可转向追踪或多段重复命中的私有技能流程
- `裂空剑气` 的释放判断应保持简单，默认仍围绕当前目标方向出手；不要把它扩成“主动寻找最佳角度”的复杂法师式范围施法逻辑
- `缚阵魔剑` 应优先复用统一的“部署型固定战斗代理体 + 定时脉冲载荷”结构，不要把魔剑做成独立 `RuntimeHero`、独立 AI 单位或可四处追人的召唤物
- 每次脉冲的拉拽应继续走共享 `ApplyForcedMovement` 语义；如果现有实现只支持“朝施法者本体拉近”，应扩成“朝技能锚点或代理体锚点拉近”的共享能力，不要为魔剑客私下写一套逐帧吸附流程
- 多目标被拉向魔剑时，必须通过共享的合法落点分配或最小间距收束处理，避免多个敌人被直接塞进同一点导致重叠、抖动或近战距离异常
- 大招不应额外附带高额范围伤害、长时间减速、不可选中或霸体；它的核心价值是阵型压缩，而不是把控制、保命和收割一次性全塞满
- 事件和日志至少要能明确记录：魔剑生成、每次脉冲触发、每个目标的拉拽生效，以及魔剑结束消失

## 设计边界

- 魔剑客的核心职责是前排压阵和制造局部多人受击窗口，不承担坦克式长期承伤职责
- 不应把魔剑客做成远距离反复放线技能的法师式消耗位；他的输出重心仍应建立在近战缠斗上
- 不应把 `裂空剑气` 扩成超长距离、高频、大宽度的主输出技能；它当前只是战士的小范围顺带 AoE
- 不应把 `缚阵魔剑` 扩成全屏群拉、逐帧持续吸附或长时间硬控链；第一阶段里它应明确保持“离散脉冲拉近”的轻量阵型压缩语义
- 不应把魔剑做成可被单独攻击、可移动或可自行索敌的第二角色；它只是战场锚点，不是独立召唤单位
- 第一阶段下，魔剑客优先验证“近战普攻 + 当前目标直线剑气 + 一局一次留场拉阵魔剑”这套带少量 AoE 的战士模板
