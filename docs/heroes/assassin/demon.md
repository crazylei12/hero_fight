# Demon

## 基础信息

- 英文名：Demon
- 中文名：恶魔
- 英雄 ID：`assassin_005_demon`
- 英雄分类：刺客
- 标签：近战、交换位移、切后排、变身、远程形态、拉扯
- 核心定位：近战扰阵型刺客，先用位置交换把当前距离自己最远的敌方英雄送到自己原位置，再把自己送进敌方后场；随后可变身为大恶魔，切换成远程输出形态并接入远程拉扯逻辑
- 默认 AI 模板：`assassin_default`
- 当前定位补充：作为第五个刺客扩展原型，不替代忍者的高频瞬移、不替代腐蚀虫的减益收割、不替代屠夫的拉拽搅阵，也不替代独行者的滚雪球自给；恶魔主要验证“强制交换双方位置 + 死亡前持续远程变身 + 形态切换后 AI 行为同步变化”这条后排交互方向

## 普攻类型

- 类型：近战瞬时命中普攻
- 释放目标：按统一 `NearestEnemy` 目标策略选择最近的合法敌方英雄；变身前不为普攻额外维护后排优先或交换目标锁头
- 效果：造成基于攻击力 `1.0` 倍率的单体伤害
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`1.35`
  - 普攻间隔：`0.88 秒`

## 小技能：Infernal Exchange（炼狱换位）

- 技能类型：单体位置交换 / 强制位移
- 释放目标：当前距离恶魔最远的合法敌方英雄，按双方中心点距离计算；若最远距离并列，则优先当前生命值更低者；若当前生命值仍并列，则优先生命值比例更低者
- 效果：
  - 施法开始时记录恶魔当前位置和目标当前位置
  - 技能结算时，恶魔移动到目标原位置，目标移动到恶魔原位置
  - 两个落点都会经过统一战场边界收束；当前第一阶段实现不额外做最小间距重排
  - 该技能不额外造成直接伤害，不附加眩晕、嘲讽、禁疗或不可选中
  - 核心收益是强行重排双方阵型：把敌方后排送到己方前场或中线附近，同时让恶魔切入敌方后场制造威胁
- 检测条件：
  - 小技能 CD 转好
  - 场上至少存在 `1` 名合法敌方目标
  - 最远合法敌人与恶魔的距离不低于 `6.75` 世界单位
  - 恶魔当前可以主动施法，且不处于硬控或其他禁止主动施法的状态
- 当前关键参数：
  - 施法距离：`全屏`，当前 `arena_stage01_flat` 可按 `56` 处理
  - 最小有效交换距离：`6.75` 世界单位
  - 交换位移时长：`0.20 秒`
  - 冷却时间：`9 秒`

## 大招：Greater Demon Form（大恶魔形态）

- 技能类型：主动型大招 / 自身变身 / 持续到死亡的运行时形态覆写
- 释放目标：自己
- 效果：
  - 恶魔变身为大恶魔，直到本次生命死亡为止
  - 变身期间，普攻从近战瞬时命中切换为远程投射物普攻
  - 变身期间，攻击力、攻击速度和攻击距离都使用远程形态参数
  - 变身期间，恶魔应被运行时 AI 视为 `usesProjectile = true` 的远程英雄，接入投射物普攻英雄的受压后撤与拉扯逻辑
  - 变身不会清除恶魔身上的负面状态，不提供回血、护盾、无敌、不可选中或硬控免疫
  - 恶魔死亡时，大恶魔形态立即结束；复活后回到基础近战形态
- 检测条件：
  - 本局尚未释放过大招
  - 当前存活且可以主动施法
  - 走当前通用大招节奏：开战前 `6 秒` 不尝试释放
  - 标准大招策略下，大致在 `10-16 秒` 后进入可释放窗口，并按通用间隔与概率检查
  - 需要恶魔处于接敌状态累计至少 `3 秒`
  - 当前不绑定 `Infernal Exchange` 成功后的固定 `2 秒` 窗口，也不额外实现低血暂缓释放门槛
- 当前关键参数：
  - 持续时间：直到恶魔死亡
  - 远程形态攻击力修正：`+45%`
  - 远程形态攻速修正：`+20%`
  - 远程形态攻击距离：`9.3` 世界单位（设计基准 `6.2`，按 Stage01 缩放后落盘）
  - 远程形态投射物速度：`13`
- 当前关键约束：
  - 这是 `主动释放型` 大招，本局仍只释放 `1 次`
  - 死亡会清除变身状态，但不会重置大招资源；复活后不能再次变身

## 落地约束

- 恶魔的小技能必须严格选择“当前距离自己最远的合法敌方英雄”，不要偷换成“敌方最后排”或“敌方最低血量后排”；如果后续需要实现，应沉淀为共享 `FarthestEnemyFromSelf` 目标策略或等价 helper
- `Infernal Exchange` 应优先实现为共享的“交换双方位置”位移效果，不应写成恶魔专属脚本；后续其他英雄若需要交换、互换或镜像位移，应能复用同一类数据契约
- 交换必须先快照双方原位置，再同时结算双方落点；不要先移动其中一方再用移动后的坐标推算另一方落点
- 当前第一阶段换位落点只要求遵守统一战场边界收束，不承诺 `1 世界单位` 最小间距避让；如果后续要强化防重叠，应升级共享强制位移落点分配，而不是只给恶魔补专属逻辑
- 交换位移本身不应伪装成眩晕、嘲讽或不可选中；如果统一强制位移流程会打断当前动作，应按共享强制位移语义处理，而不是给恶魔另开例外
- `Greater Demon Form` 应作为运行时形态覆写表达，不应直接改写 `HeroDefinition`、`HeroStatsData` 或静态 `BasicAttackData`
- 远程形态必须让 AI 和普攻系统同时看到形态变化：普攻使用投射物、攻击距离改变，且 `usesProjectile = true` 触发远程受压后撤逻辑
- 变身持续到死亡属于本英雄大招特例；当前变身开始通过大招释放体现，死亡会清除运行时形态，复活后回到近战形态，且大招资源不重置

## 设计边界

- 恶魔的核心职责是无耻地交互敌方后排、打乱阵型并在敌阵附近切换远程输出形态，不承担坦克式正面承伤或团队保护职责
- 不应把 `Infernal Exchange` 扩成可手动点选任意目标、可连续交换多名敌人或附带高额伤害的全能先手技
- 不应把大恶魔形态扩成 `远程形态 + 无敌 + 免控 + 回血` 的规则特权；它只改变输出形态和相关属性
- 不应让大恶魔形态在死亡后继续保留，也不应让复活后再次免费变身
- 不应把恶魔做成开局就在后排安全站桩的纯射手；它必须先承担近战刺客交换切入的站位风险
- 第一阶段下，恶魔优先验证“近战普攻 + 最远敌人位置交换小技能 + 一局一次死亡前持续远程变身 + 变身后远程拉扯 AI”这套后排交互刺客模板
