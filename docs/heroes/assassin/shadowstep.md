# Shadowstep

## 基础信息

- 英文名：Shadowstep
- 中文名：忍者
- 英雄 ID：`assassin_001_shadowstep`
- 英雄分类：刺客
- 标签：近战、突进、切后排、收割
- 核心定位：近战切后排刺客，依靠瞬移稳定贴到敌方最后排，用一局一次的不可选中窗口在后排区域连续换位追杀
- 默认 AI 模板：`assassin_default`

## 普攻类型

- 类型：近战瞬时命中普攻
- 释放目标：默认优先攻击当前已切入到身边的敌方目标；没有合法近身目标时，继续前压并优先寻找敌方后排脆皮
- 效果：造成基于攻击力 `1.0` 倍率的单体伤害
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`1.3`
  - 普攻间隔：`0.83 秒`

## 小技能：Shadow Blink

- 技能类型：瞬移切入 + 单体斩击
- 释放目标：当前存活敌方英雄中“最后排”的那一个；“最后排”定义为距离其本方出生点最近的英雄，若并列则优先生命值比例更低者，再优先当前生命值更低者
- 效果：
  - 瞬移到目标身边的合法近身落点
  - 落地后立即对目标打出一次视为强化普攻的近战斩击
  - 该斩击沿用近战命中与暴击规则，但伤害倍率高于普通普攻
  - 技能核心收益来自稳定切到敌方后排，而不是提供控制或范围爆发
- 检测条件：
  - 小技能 CD 转好
  - 场上存在合法敌方目标
  - 能找到合法的目标近身落点
- 当前关键参数：
  - 施法距离：`全屏`，当前 `arena_stage01_flat` 可按 `40` 处理
  - 落点距离：目标身边 `0.9`
  - 斩击伤害倍率：基于攻击力 `1.25`
  - 冷却时间：`6 秒`

## 大招：Smoke Veil

- 技能类型：自身增益 Buff / 不可选中 + 小技能 CD 覆写
- 释放目标：自己
- 效果：
  - 立即为自己附加 `6 秒` 的 `不可选中`
  - 状态期间，忍者仍留在战场上，可以移动、普攻和继续释放小技能
  - 状态期间，小技能冷却时间固定覆写为 `1 秒`
  - 若开大瞬间小技能剩余 CD 高于 `1 秒`，则立刻压到 `1 秒`
  - `不可选中` 不等于 `无敌`；忍者仍可被非指向范围效果、持续伤害和控制命中
- 检测条件：
  - 本局尚未释放过大招
  - 当前存在合法的“最后排敌方英雄”
  - 优先在自身生命值比例不高于 `60%`，或该目标生命值比例不高于 `70%` 时释放
  - 若开战 `35 秒` 后仍未满足，则只要存在合法目标即可释放
- 当前关键参数：
  - 不可选中持续时间：`6 秒`
  - 大招期间小技能冷却：`1 秒`
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 当前工程中的 `assassin_001_shadowstep` 仍是占位实现；后续应直接沿用这个稳定英雄 ID，把占位技能方案替换为本文档定义的 `Shadow Blink / Smoke Veil`
- `Shadow Blink` 应优先复用统一的 `瞬移 reposition + 单次近战命中` 组合表达，不应把“闪到目标身边并打一下”拆成刺客专属移动脚本
- `Shadow Blink` 的起点消失 / 终点显形反馈，应优先复用共享“角色残影快照 + 到达后短暂淡入”的表现逻辑，而不是单独维护忍者专属闪现特效脚本
- “敌方最后排英雄”应尽量沉淀为可复用的目标选择策略或统一 helper，不要只在 `Shadowstep` 的技能流程里偷偷硬编码
- `Smoke Veil` 必须直接复用正式 `Untargetable` 状态语义；它不应把英雄从战场列表里移除，也不应顺便变成无敌、免控或完全失去碰撞的相位化状态
- “大招期间小技能 CD = 1 秒”应优先作为临时冷却覆写或运行时 Buff 表达，而不是把技能资产本体的静态 CD 改坏

## 设计边界

- 忍者的核心职责是切入后排、追击残血和制造后排压力，不承担前排承伤职责
- 不应把忍者做成依赖大范围伤害或连续硬控吃饭的爆发法师式角色
- 不应把忍者的大招扩成 `不可选中 + 无敌 + 免控` 的全能保命按钮
- 不应把忍者做成持续站桩输出的远程刺客
- 第一阶段下，忍者优先验证“全屏切后排小技能 + 一局一次不可选中窗口 + 频繁追击后排”这套刺客模板
