# Butcher

## 基础信息

- 英文名：Butcher
- 中文名：屠夫
- 英雄 ID：`assassin_003_butcher`
- 英雄分类：刺客
- 标签：近战、强制位移、减疗、搅阵
- 核心定位：近战搅阵型刺客，不靠自己深跳后排，而是用钩子把敌方最远单位强行拖进前线，再用一局一次的群体拉拽大招把敌方阵型整体扯乱并压出禁疗窗口
- 默认 AI 模板：`assassin_default`
- 当前定位补充：作为 `assassin_001_shadowstep` 和 `assassin_002_tidefin` 之后的第三个刺客扩展原型，不替代忍者的高机动切入模板，也不替代腐蚀虫的持续削弱模板，主要验证“后排定点拉拽 + 目标侧禁疗 + 群体强制聚拢”这条搅阵型刺客方向

## 普攻类型

- 类型：近战瞬时命中普攻
- 释放目标：按统一 `NearestEnemy` 目标策略选择最近的合法敌方英雄；屠夫不为普攻额外维护后排优先或被钩中目标专属锁头，钩子把目标拖到身前后，由最近目标规则自然接管后续追击
- 效果：造成基于攻击力 `1.05` 倍率的单体伤害
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`1.4`
  - 普攻间隔：`0.91 秒`

## 小技能：Gore Hook

- 技能类型：单体拉拽 + 受治疗率下降 Debuff
- 释放目标：优先按共享 `BackmostEnemy` 目标策略锁定敌方当前最后排、通常也是距离自己最远的合法敌方英雄；若当前没有明显后排目标，则回退到最近的合法敌方英雄
- 效果：
  - 向目标发出一段单体钩链命中表现
  - 命中后将目标沿施法者与目标的连线方向强制拉到施法者身前的合法近身落点
  - 目标落点默认位于施法者前方约 `1.15` 的位置，若该点被占用，则回退到施法者身边最近的合法落点
  - 命中后为目标附加一次 `HealTakenModifier`
  - 当前默认减疗幅度为 `-100%`
  - 减疗持续 `4 秒`
  - 该技能当前不附带额外直接伤害；核心收益来自拆阵、抓后排和切断对方治疗回补
- 检测条件：
  - 小技能 CD 转好
  - 场上存在合法敌方目标
- 当前关键参数：
  - 施法距离：`全屏`，当前 `arena_stage01_flat` 可按 `56` 处理
  - 拉拽落点距离：施法者身前 `1.15`
  - 强制位移时长：`0.35 秒`
  - 禁疗持续时间：`4 秒`
  - 冷却时间：`8 秒`

## 大招：Carnage Reel

- 技能类型：全敌方拉拽 + 群体受治疗率下降 Debuff
- 释放目标：所有合法敌方英雄
- 效果：
  - 同时向所有敌方英雄发出钩链命中表现
  - 对每个目标分别结算一次 `TowardSource` 的强制位移，将其拉向屠夫身边
  - 系统会优先为每个目标分配屠夫周围 `1.2 ~ 2.2` 的合法落点，而不是全部叠在同一个坐标点
  - 若某个目标原本已经比该分配环更靠近屠夫，则保持 `TowardSource` 只拉近、不反向外推的约束，不会为了凑环形站位把目标再推出去
  - 对每个命中的目标附加一次 `HealTakenModifier`
  - 当前默认减疗幅度为 `-100%`
  - 减疗持续 `4 秒`
  - 大招当前不附带直接伤害、硬控或无敌；它的核心强度来自“全队搅阵 + 集体禁疗”
- 检测条件：
  - 本局尚未释放过大招
  - 优先在至少 `3` 名敌方英雄存活时释放
  - 若敌方有 `2` 名及以上英雄生命值比例不高于 `70%`，则显著提高释放优先级
  - 若开战 `40 秒` 后仍未满足上述理想窗口，则只要仍有至少 `2` 名敌方英雄存活即可释放
- 当前关键参数：
  - 拉拽目标数：所有合法敌方英雄
  - 单目标落点半径：屠夫周围 `1.2 ~ 2.2`
  - 强制位移时长：`0.45 秒`
  - 禁疗持续时间：`4 秒`
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 屠夫的“钩子”在第一阶段应优先落成 `自动锁定目标 + ApplyForcedMovement(TowardSource)` 的统一技能表达，不扩成玩家手动瞄准、路径碰撞命中或可勾中途单位的 MOBA 式技能射线系统
- `Gore Hook` 应优先复用共享 `BackmostEnemy` 目标策略；如果现有实现与“当前最远敌人”不完全等价，优先做成可复用的共享目标策略修正，不要只给屠夫私下塞一段孤立索敌分支
- 屠夫的普攻索敌继续沿用统一 `NearestEnemy`，不要因为它是刺客或因为目标刚被拉到身前，就再额外补一套屠夫专属普攻锁头逻辑
- 单体拉拽与群体拉拽都必须复用统一 `ApplyForcedMovement` 的 `TowardSource` 语义，不新增“钩中状态”“被拖拽状态”这类只服务单英雄的新底层状态类型
- `HealTakenModifier` 必须作为正式统一状态接入统一治疗入口，同时影响直接治疗和持续治疗；屠夫技能命中后不应靠手写 if 分支去临时拦截治疗
- 群体拉拽的多目标落点必须通过共享的合法落点分配或最小间距收束处理，避免多个敌人被直接塞进同一点导致重叠、抖动或后续普攻距离异常

## 设计边界

- 屠夫的核心职责是打乱敌方站位、抓后排和压禁疗窗口，不承担坦克式正面开团、长时间承伤或稳定硬控链职责
- 不应把屠夫做成带高额范围爆发、群体眩晕和群体嘲讽的全能开团核心；他的强度重点应放在阵型破坏，而不是自己一键打满伤害
- 不应把屠夫的小技能扩成可无限远手动点射、可被小兵或中途目标拦截的复杂技能判定，这会显著拖慢第一阶段原型
- 不应把屠夫做成安全后排站桩输出位；他仍然必须承担近战刺客贴身追杀的站位风险
- 不应把大招扩成“全体拉拽 + 高伤害 + 长时间硬控 + 长时间禁疗”的全收割技能；在第一阶段里，它应优先验证群体搅阵和禁疗这两个核心语义
