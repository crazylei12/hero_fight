# Shieldwarden

## 基础信息

- 英文名：Shieldwarden
- 中文名：护阵者
- 英雄 ID：`tank_002_shieldwarden`
- 英雄分类：坦克
- 标签：近战、承伤、控制、反切入
- 核心定位：反切入近战坦克，优先把突入己方阵线的刺客拉到自己身上，用单点嘲讽拆对方关键切入，再用全体嘲讽和自身防御爆发窗口替队友扛住一波集火
- 默认 AI 模板：`tank_default`
- 当前定位补充：作为 `tank_001_ironwall` 之后的第二个坦克扩展原型，不替代骑士的团队减伤稳团模板，主要验证“按职业优先的单体嘲讽 + 全体嘲讽 + 自身高防御窗口”这条反切入坦克方向

## 普攻类型

- 类型：近战瞬时命中普攻
- 释放目标：当前索敌目标；坦克默认优先接敌并黏住最近敌人，找不到时继续向敌方区域前压
- 效果：造成基于攻击力 `1.0` 倍率的单体伤害
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`1.8`
  - 普攻间隔：`1.05 秒`

## 小技能：Warden's Call

- 技能类型：单体控制 / 嘲讽
- 释放目标：施法距离 `8` 内的单个敌方英雄；优先选择敌方 `刺客`，若有多个则优先当前距离更近者；若当前没有合法刺客目标，则退回最近敌方英雄
- 效果：
  - 对目标附加 `1.5 秒` 嘲讽
  - 嘲讽期间目标会被迫把护阵者视为当前目标，并停止主动施法
  - 当前参数下，这个技能主要用于把敌方刺客或先手切入单位从己方后排身上拉回自己，不承担主要伤害职责
- 检测条件：
  - 小技能 CD 转好
  - 施法距离 `8` 内存在合法敌方目标
- 当前关键参数：
  - 施法距离：`8`
  - 嘲讽持续时间：`1.5 秒`
  - 冷却时间：`8 秒`

## 大招：Last Bastion

- 技能类型：全体控制 + 自身增益 Buff / 属性上升
- 释放目标：全体敌方英雄，同时对自己生效一层防御增益
- 效果：
  - 立即对所有存活敌方英雄附加 `3 秒` 嘲讽
  - 同时为自己附加一次 `DefenseModifier`
  - 防御力提升为 `+250%`
  - 防御增益持续 `4 秒`
  - 大招核心收益是强行把敌方整波火力和技能节奏拉到自己身上，为己方后排争取重新站位和输出空间
- 检测条件：
  - 本局尚未释放过大招
  - 优先要求自身 `6` 范围内至少存在 `3` 名敌方英雄
  - 若其中至少 `1` 名为刺客，则显著提高释放优先级
  - 若开战 `45 秒` 后仍未满足，则只要场上仍有至少 `2` 名存活敌方英雄，且自身生命值比例不高于 `70%` 即可释放
- 当前关键参数：
  - 全体嘲讽持续时间：`3 秒`
  - 自身防御力提升：`+250%`
  - 自身防御增益持续时间：`4 秒`
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- `Warden's Call` 的“优先刺客，否则退回最近敌人”应优先沉淀为共享的“职业过滤 + 回退目标策略”，不要只在护阵者流程里单独硬编码
- 小技能和大招的嘲讽都必须直接复用统一 `Taunt` 状态语义：强制把来源视为当前目标，并禁止主动释放技能；不要顺手扩成完整仇恨系统
- `Last Bastion` 的“嘲讽敌方所有”应通过一次施法对每个存活敌方单位分别附加正式嘲讽状态来表达，不要额外创建“全场进入强制攻击模式”的专属战斗状态
- 大招的自身防御提升必须走统一属性 Buff 入口，不要把“开大后更硬”散写在伤害结算特判里
- 日志和事件至少要能明确记录：护阵者释放大招、命中的敌人数、各目标被嘲讽的持续时间，以及自身防御 Buff 的开始和结束

## 设计边界

- 护阵者的核心职责是反切入、承伤和集火转移，不承担爆发收割职责
- 不应把护阵者做成拥有高机动突进或切后排能力的主动开团坦克
- 不应把护阵者做成兼具全队防御 Buff、护盾、治疗和无敌的全能保命核心
- 不应把护阵者的大招扩成超长时长的全场控制链；它更应该是一段短而强的吸火窗口
- 第一阶段下，护阵者优先验证“近战普攻 + 职业优先单体嘲讽 + 全体嘲讽并强化自身防御”这套反切入坦克模板
