# Ironwall

## 基础信息

- 英文名：Ironwall
- 中文名：骑士
- 英雄 ID：`tank_001_ironwall`
- 英雄分类：坦克
- 标签：近战、承伤、保护、团队增益
- 核心定位：前排近战保护坦克，优先顶在己方前线，用短时范围伤害分担替附近队友承压，再用全队防御增益稳住团战
- 默认 AI 模板：`tank_default`

## 普攻类型

- 类型：近战瞬时命中普攻
- 释放目标：当前索敌目标；坦克默认优先接敌并黏住最近敌人，找不到时继续向敌方区域前压
- 效果：造成基于攻击力 `1.0` 倍率的单体伤害
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`1.8`
  - 普攻间隔：`1.11 秒`

## 小技能：Bulwark Bond（壁垒共担）

- 技能类型：范围保护 / 伤害分担
- 释放目标：以自己为施法中心；施法时收集自身 `4.5` 范围内的其他友方英雄，不包含自己
- 效果：
  - 立即对范围内命中的每个其他友军附加一层 `分担伤害`
  - 状态持续期间，这些友军每次在自身护盾结算后的实际受伤中，会有一部分转由骑士承担
  - 骑士承担的转移伤害仍继续走自己的统一受伤入口，因此会正常受自身护盾、防御、无敌和击杀归属规则影响
  - 当前参数下，这个技能更偏向“抱团承压窗口”，让骑士替邻近队友顶住一波前压或爆发
- 检测条件：
  - 小技能 CD 转好
  - 自身 `4.5` 范围内至少存在 `1` 名其他存活友军
  - 这些友军中至少 `1` 名距离任意敌方英雄 `6` 以内
- 当前关键参数：
  - 施法半径：`4.5`
  - 伤害分担比例：`35%`
  - 持续时间：`4 秒`
  - 冷却时间：`4 秒`

## 大招：Iron Oath

- 技能类型：全体增益 Buff / 属性上升
- 释放目标：全体友方英雄，包括自己
- 效果：
  - 立即为全体友军附加统一的防御力提升状态
  - Buff 直接附着在友军单位身上，不依赖固定区域，会随友军后续移动持续生效
  - 当前参数下更偏向“团队减伤窗口”，不提供护盾、治疗或无敌
- 检测条件：
  - 本局尚未释放过大招
  - 当前至少 `3` 名存活友军距离任意敌方英雄 `6` 以内
  - 若开战 `45 秒` 后仍未满足，则只要求至少 `2` 名存活友军已进入上述交战距离
- 当前关键参数：
  - 防御力提升：`+200%`
  - 持续时间：`8 秒`
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- `Bulwark Bond` 必须直接复用统一 `分担伤害` 状态语义：状态附着在被保护友军身上，`source` 指向骑士，不新增专属“连线承伤”小系统
- 为保持第一阶段实现简单，受保护目标应按施法瞬间一次性收集，不要求额外维护“持续进出范围就动态补上或移除”的光环逻辑
- 分担比例的基准必须继续使用“友军护盾结算后的实际伤害”，不能回退到护盾前毛伤害，也不能把已经分担出去的伤害再次继续向外连锁
- 日志和事件至少要能明确记录：骑士释放技能、命中的友军数量、每个目标的分担比例与持续时间，以及后续由分担伤害转入骑士的受伤记录

## 设计边界

- 骑士的核心职责是前排承伤、限制推进和团队保护，不承担爆发收割职责
- 不应把骑士做成靠高机动切后排的突进型英雄
- 不应把骑士做成以大范围连续硬控为主的控制核心
- 不应把骑士的小技能扩成同时附带高比例分担、群体护盾、治疗和强嘲讽的全能保人按钮
- 不应把骑士的大招扩成护盾、治疗或无敌类全能保命技能
- 第一阶段下，骑士优先验证“近战瞬时普攻 + 范围伤害分担小技能 + 全队防御 Buff 大招”这套稳团保护型坦克模板
