# Mundo

## 基础信息

- 英文名：Mundo
- 中文名：蒙多
- 英雄 ID：`tank_004_mundo`
- 英雄分类：坦克
- 标签：近战、承伤、自愈、站场
- 核心定位：纯承伤近战坦克，不靠控制或显式保护队友建立价值，而是靠常驻自愈和一局一次的爆发续命长时间堵在正面战线；当敌方缺少越过前排的能力时，他会把推进拖得非常难受，但一旦对手能绕后或挂上减疗，他的存在感就会明显下滑
- 默认 AI 模板：`tank_default`
- 当前定位补充：作为 `tank_001_ironwall`、`tank_002_shieldwarden` 和 `tank_003_tidehunter` 之后的第四个坦克扩展原型，不替代骑士的团队保护、不替代护阵者的反切入嘲讽，也不替代潮汐猎人的范围反开；主要验证“无强控、无显式保人、纯靠站场和自愈扛线”的坦克方向

## 当前数值倾向

- 基础承伤面板可以略高于 `Ironwall` / `Tidehunter` 一类主坦，但不要高到完全脱离 `docs/planning/stage-01-ultimate-numeric-guidelines.md` 中现有的坦克锚点
- 蒙多的强度预算应优先放在“持续自愈能否让他多站几秒”上，而不是额外补控制、突进、护盾或团队保护
- 若后续实装发现仅靠更高基础生命 / 防御就已经过硬，应优先回收面板，再保留技能自愈辨识度

## 普攻类型

- 类型：近战瞬时命中普攻
- 释放目标：当前索敌目标；坦克默认优先接敌并黏住最近敌人，找不到时继续向敌方区域前压
- 效果：造成基于攻击力 `1.0` 倍率的单体伤害；普攻本体不承担控制、减疗或保护职责
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`1.9`
  - 普攻间隔：`1.11 秒`

## 小技能：Brute Metabolism

- 技能类型：被动型小技能 / 分段自我持续治疗
- 释放目标：自己；常驻生效，无主动施放动作
- 效果：
  - 每隔 `1 秒`，对自己结算一次基于最大生命值的自我治疗
  - 当当前生命值比例 `> 60%` 时，每次恢复 `1.5%` 最大生命值
  - 当当前生命值比例处于 `30% ~ 60%` 时，每次恢复 `2.8%` 最大生命值
  - 当当前生命值比例 `<= 30%` 时，每次恢复 `4.5%` 最大生命值
  - 这条被动的核心收益是“越被压低越难被轻易磨死”，不是让蒙多在脱战后瞬间回满
- 检测 / 生效条件：
  - 蒙多处于存活状态
  - 每次治疗都按当下生命值比例重新判断所在回复档位
  - 死亡后停止，复活后重新按当前生命值比例恢复生效
- 当前关键参数：
  - 治疗跳动间隔：`1 秒`
  - 高血量回复：`1.5% 最大生命值 / 秒`
  - 中血量回复：`2.8% 最大生命值 / 秒`
  - 低血量回复：`4.5% 最大生命值 / 秒`

## 大招：Monstrous Recovery

- 技能类型：自我持续治疗 / 生存爆发
- 释放目标：自己
- 效果：
  - 释放后进入 `6 秒` 的高速恢复期
  - 持续期间每隔 `0.5 秒`，对自己结算一次 `6% 最大生命值` 的治疗
  - 大招不提供护盾、无敌、不可选中、防御 Buff 或控制效果；它的价值只在于把蒙多继续留在前排
  - 若在正面承压中完整吃满，大招加上小技能被动的总回复应接近一条标准坦克生命线，但只要遭遇明显减疗，收益就会大幅下滑
- 检测条件：
  - 本局尚未释放过大招
  - 当前生命值比例 `<= 45%`
  - 自身 `6.5` 范围内至少存在 `2` 名敌方英雄
  - 若当前生命值比例已经 `<= 25%`，则允许在自身 `6.5` 范围内仅有 `1` 名敌方英雄时直接释放
  - 若开战 `45 秒` 后仍未满足理想窗口，则生命值阈值放宽为 `<= 60%`
- 当前 Stage01 代码落地说明：
  - 受共享大招决策模板表达能力限制，当前实现采用近似规则：默认要求 `<= 45%` 血量且 `6.5` 范围内至少 `2` 名敌人；开战 `45 秒` 后放宽为 `<= 60%` 血量且至少 `1` 名敌人
  - 文档中的“`<= 25%` 血量时允许 `1` 名敌人直接释放”仍保留为后续扩展方向，但本轮没有为蒙多额外加入英雄私有决策分支
- 当前关键参数：
  - 持续时间：`6 秒`
  - 治疗跳动间隔：`0.5 秒`
  - 单次治疗量：`6% 最大生命值`
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- `Brute Metabolism` 应优先复用共享的“按当前生命值比例分段结算被动收益”能力，不要单独写一个 `MundoOnlyUpdate` 风格的私有回血脚本
- 小技能与大招的所有回复都必须走统一治疗入口，并明确受目标侧 `HealTakenModifier / 受治疗率修正` 影响；`-1.0` 的禁疗应能完全阻断这些回复
- 大招应优先表达为正式的自我持续治疗序列或等价的统一 `HealOverTime` 载荷，不要直接在英雄脚本里绕过日志和事件去改当前生命值
- 蒙多当前定位的代价就是缺乏控制和保人能力；不要为了“看起来更有用”再顺手给他补嘲讽、击飞、护盾、团队减伤或切后排位移
- 事件与日志至少应能明确记录：被动当前处于哪个回复档位、大招开始和结束、每次实际回复量，以及减疗导致的回复衰减

## 设计边界

- 蒙多的核心职责是前排站桩承伤和拖线，不承担反切入、团控或团队保护职责
- 不应把蒙多做成拥有嘲讽、击飞、强位移或突后排能力的开团坦克
- 不应把蒙多做成兼具高额自愈、护盾、无敌、分担伤害和全队防御 Buff 的全能生存核心
- 不应把蒙多的大招扩成解除控制、免疫减疗或越打越强的全能保命按钮；敌方减疗应该始终是他的明确反制点
- 如果敌方已经能稳定越过前排攻击己方后排，蒙多应显得相对笨重和无能为力；这不是缺陷，而是这个原型的应有弱点
- 第一阶段下，蒙多优先验证“近战普攻 + 低血增强自愈被动 + 一局一次自我爆发回血大招”这套纯肉前排坦克模板
