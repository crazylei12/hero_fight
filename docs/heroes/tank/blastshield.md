# Blastshield

## 基础信息

- 英文名：Blastshield
- 中文名：爆破盾牌
- 英雄 ID：`tank_005_blastshield`
- 英雄分类：坦克
- 标签：近战、承伤、反近战、减疗、击退、陷阱
- 核心定位：反近战前排坦克，通过持盾姿态惩罚贴身攻击自己的敌方近战英雄，把对方前排和刺客不断打退、压低回复收益；但他缺少远距离开团、远程反制和追击后排能力，面对远程站桩输出时只能依靠队友或地雷区限制走位
- 默认 AI 模板：`tank_default`
- 当前定位补充：作为 `tank_001_ironwall`、`tank_002_shieldwarden`、`tank_003_tidehunter` 和 `tank_004_mundo` 之后的第五个坦克扩展原型，不替代骑士的团队保护、护阵者的嘲讽反切入、潮汐猎人的范围反开或蒙多的纯自愈站场；主要验证“持盾期间牺牲攻击换取近战受击反制 + 一局一次前方固定地雷区”这条反近战坦克方向

## 当前数值倾向

- 基础承伤面板应接近标准主坦，但不要高到脱离现有坦克锚点；他的额外硬度主要来自持盾窗口
- 攻击力可以偏低，常态普攻不是主要输出来源
- 移速保持坦克标准偏慢，持盾期间再大幅下降，避免他一边高速追人一边稳定反制
- 强度预算应优先放在“近战打他会付出代价”和“地雷区让敌方近战前压不舒服”上，不应额外补远程开火、突进、嘲讽、治疗或团队保护

## 建议实现基线

- 最大生命：`550`
- 攻击力：`34`
- 防御力：`38`
- 攻速：`0.9`，约等于 `1.11 秒 / 次`
- 移速：`4.0`
- 暴击率：`5%`
- 暴击伤害：`1.5`
- 攻击距离：`1.8`

## 普攻类型

- 类型：近战瞬时命中普攻
- 释放目标：当前索敌目标；坦克默认优先接敌并黏住最近敌人，找不到时继续向敌方区域前压
- 效果：造成基于攻击力 `1.0` 倍率的单体伤害；普攻本体不附带控制、减疗或反伤，核心特色留给持盾反制和大招地雷
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`1.8`
  - 普攻间隔：`1.11 秒`

## 小技能：Shield Brace（持盾）

- 技能类型：主动型小技能 / 自身姿态覆写 / 近战受击反制
- 释放目标：自己
- 效果：
  - 爆破盾牌进入 `4 秒` 持盾姿态
  - 持盾期间不进行普攻，且默认不主动释放其他技能；这段时间应视为正式动作覆写，不把“无法攻击”扩成可施加给敌人的通用缴械状态
  - 持盾期间获得 `DefenseModifier`
  - 持盾期间获得 `MoveSpeedModifier`
  - 当持盾期间受到敌方近战英雄的有效直接命中时，对该攻击来源触发一次盾击反制
  - 盾击反制对攻击来源造成一次基于爆破盾牌攻击力的直接伤害
  - 盾击反制为攻击来源附加一次 `HealTakenModifier`
  - 盾击反制将攻击来源沿远离爆破盾牌的方向击退一小段距离
  - 同一攻击来源在短时间内重复命中时，应受每来源触发间隔限制，避免高攻速或多段近战命中让反制无限连跳
- 近战命中判定：
  - 默认只响应敌方 `usesProjectile = false` 的近战普攻
  - 若后续某些技能明确标记为“近身直接打击”，也可以纳入这套反制入口
  - 远程投射物普攻、持续伤害、持续区域 tick、地雷爆炸和非接触型范围技能不触发持盾反制
- 检测条件：
  - 小技能 CD 转好
  - 自身周围 `3.0` 范围内存在至少 `1` 名敌方近战英雄
  - 或最近 `1.5 秒` 内自身受到过敌方近战英雄的有效直接命中
  - 如果当前威胁主要来自远程英雄且没有近战压近，释放优先级应明显降低
- 当前关键参数：
  - 持盾持续时间：`4 秒`
  - 防御力提升：`+180%`
  - 移速降低：`-65%`
  - 反制伤害：基于攻击力 `0.75`
  - 减疗幅度：`-50%`
  - 减疗持续时间：`3 秒`
  - 击退距离：`1.8`
  - 强制位移时长：`0.25 秒`
  - 同一攻击来源触发间隔：`0.8 秒`
  - 冷却时间：`8 秒`

## 大招：Blast Minefield（爆破雷阵）

- 技能类型：部署型固定触发点 / 前方地雷区 / 一次性范围伤害
- 释放目标：记录施法瞬间的前方方向；若当前存在合法敌方目标，则以前往该目标的方向作为布雷方向，否则默认朝敌方半场方向布雷
- 效果：
  - 爆破盾牌向前方散布一片固定地雷
  - 每颗地雷都是固定触发点，不移动、不索敌、不作为英雄单位，默认不可选中、不可受伤、无碰撞阻挡
  - 地雷会一直保留，直到有敌方英雄进入触发半径后爆炸，或本局战斗结束
  - 地雷不因爆破盾牌死亡或进入复活等待而提前消失；伤害、日志和击杀归属仍挂回爆破盾牌与本次大招
  - 每颗地雷只爆炸一次，爆炸后从战场移除
  - 地雷爆炸时，对爆炸半径内所有敌方英雄造成一次范围伤害
  - 大招本身不附带减疗、击退、眩晕或嘲讽；它的核心价值是让敌方近战前压路径变差，而不是变成全能控制大招
- 随机布雷规则：
  - 每次释放时，在施法者前方 `2.5 ~ 8.0` 的距离内随机散布 `7` 颗地雷
  - 随机区域宽度约 `5.5`
  - 每颗地雷的最终点位在施法瞬间一次性确定，不随敌人移动或施法者后续朝向变化重新随机
  - 地雷之间应尽量保持至少 `2.0` 的间距，避免一次接敌稳定连爆过多；如果随机候选点过近，应重抽或回退到最近合法点
  - 随机布点必须走统一随机数服务，保证同一 seed 下可复现；不要在表现层临时随机出一套只影响画面的假点位
- 检测条件：
  - 本局尚未释放过大招
  - 当前存在合法敌方目标
  - 释放逻辑保持宽松：不再硬性要求前方预计命中人数或敌方近战数量
  - 若自身周围 `6.5` 范围内已有敌方近战英雄压近，可以提高释放优先级，但这不是释放硬门槛
  - 如果敌方存活单位主要是远程且没有压进前线，可以降低释放优先级，但不应让大招长期卡住不放
- 当前关键参数：
  - 地雷数量：`7`
  - 地雷触发半径：`1.0`
  - 爆炸半径：`2.0`
  - 单颗地雷伤害：基于攻击力 `2.4`
  - 地雷持续方式：触发前持续到战斗结束
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 爆破盾牌当前定位为“第五个坦克扩展原型”，不替代既有坦克方向；后续实现时不要为了补短板把他扩成全能前排
- `Shield Brace` 的持盾姿态应优先表达为技能执行期间的正式动作覆写，并通过统一状态系统附加 `DefenseModifier` 与 `MoveSpeedModifier`
- 持盾期间“无法攻击”只约束爆破盾牌自己，不应为了这个英雄新增通用 `缴械` 状态；第一阶段状态文档当前明确暂不把缴械纳入底层正式集合
- 近战受击反制必须明确依赖“敌方近战直接命中”入口，不要让远程投射物、DOT、持续区域或地雷爆炸也触发反制
- 反制伤害继续走统一伤害结算；减疗继续走统一 `HealTakenModifier`；击退继续走共享 `ApplyForcedMovement`，不要新增“盾反伤害”“盾反减疗”“盾反击退”这类只服务单英雄的新底层
- 每来源触发间隔应作为正式运行时数据处理，不要把它藏在表现层或动画事件里
- `Blast Minefield` 应优先复用或扩展统一的“部署型固定战斗代理体 / 固定触发点”能力，表达为“接近触发 + 一次性范围伤害”的共享载荷
- 地雷不应做成独立 `RuntimeHero`、独立 AI 单位、可阻挡单位的实体或独立计分来源
- 地雷的触发、爆炸、移除、命中目标数和实际伤害都应进入统一事件与日志，方便后续复查地雷是否真的只在敌方进入范围时触发
- 大招地雷需要有可读的地面提示或表现资源；表现层只能显示地雷和爆炸范围，不能反向决定触发半径、爆炸半径或持续方式

## 设计边界

- 爆破盾牌的核心职责是惩罚敌方近战前压、打散贴身阵型和压低近战单位回复收益，不承担远程压制、后排切入或全队保护职责
- 不应给爆破盾牌补远距离突进、钩子、全场嘲讽、远程反击或反射远程弹道；远程英雄能相对安全地消耗他，是这个英雄必须保留的反制点
- 不应把持盾做成永久高覆盖率姿态；持盾必须有明显空窗，且持盾期间不能同时保持常规输出
- 不应把持盾反制扩成对所有伤害来源都生效的万能反伤；它只服务“敌方近战打上来会很难受”这个核心主题
- 不应把大招地雷做成隐形、全屏、无法规避或带长控制链的必杀陷阱；地雷应是前方路径封锁和阵地压迫，而不是稳定秒杀远程后排
- 不应让地雷参与默认索敌、最小间距、body blocking、出生/复活或击杀计分语义；它只是一次性触发点
- 第一阶段下，爆破盾牌优先验证“近战普攻 + 持盾反近战姿态 + 一局一次前方固定地雷区”这套反近战坦克模板
