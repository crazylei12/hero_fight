# Tidehunter

## 基础信息

- 英文名：Tidehunter
- 中文名：潮汐猎人
- 英雄 ID：`tank_003_tidehunter`
- 英雄分类：坦克
- 标签：近战、承伤、控制、反开、团控
- 核心定位：前排团战控场坦克，不靠嘲讽强拉目标，而是越被多人正面围攻越难被压垮；当敌方压进自己周围后，再用往返扩散的海潮把整片前线分两段抬起，打断推进和收割节奏
- 默认 AI 模板：`tank_default`
- 当前定位补充：作为 `tank_001_ironwall` 和 `tank_002_shieldwarden` 之后的第三个坦克扩展原型，不替代骑士的团队减伤稳团模板，也不替代护阵者的反切入嘲讽模板，主要验证“被集火实时增防 + 以自身为圆心的双程范围击飞大招”这条更偏团战反开与正面控场的坦克方向

## 普攻类型

- 类型：近战瞬时命中普攻
- 释放目标：当前索敌目标；坦克默认优先接敌并黏住最近敌人，找不到时继续向敌方区域前压
- 效果：造成基于攻击力 `1.0` 倍率的单体伤害
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`1.9`
  - 普攻间隔：`1.11 秒`

## 小技能：Undertow Carapace（逆潮甲壳）

- 技能类型：被动型小技能 / 动态防御力增益
- 释放目标：自己；常驻生效，无主动施放动作
- 效果：
  - 潮汐猎人维护一个短时“承压来源窗口”，用来估算当前有多少不同敌方英雄正在集火自己
  - 最近 `1.5 秒` 内，每有 `1` 名不同的敌方英雄对潮汐猎人造成过 `直接伤害`，就为自己提供一层防御加成
  - 每层防御力提升 `+20%`
  - 最多计入 `4` 名不同来源，总上限为 `+80%`
  - 同一来源重复命中只刷新自己的持续时间，不额外增加层数
  - `DOT / 持续伤害` 不用于新建或刷新层数，避免被离场伤害长期挂住被动
  - 该技能的核心收益是“越被多人正面围殴越硬”，不是提供护盾、减伤或无敌
- 检测 / 生效条件：
  - 潮汐猎人处于存活状态
  - 承压来源集合变化时应立刻刷新防御加成
  - 死亡后清空，复活后重新按新的受击来源开始累计
- 当前关键参数：
  - 承压来源统计窗口：`1.5 秒`
  - 每名来源提供防御力：`+20%`
  - 最多计入来源数：`4`
  - 防御力总上限：`+80%`
  - 生效方式：`按不同来源实时刷新`

## 大招：Tidal Rebound（怒潮回卷）

- 技能类型：以自身为圆心的双程范围伤害 + 分层范围 `KnockUp`
- 释放目标：以自己为圆心释放一圈海潮；海潮会先向外扩散到最大半径，再从外圈回卷到中心点，命中过程中只影响敌方英雄
- 效果：
  - 施法瞬间记录一个固定中心点，并从该点发出一段向外扩张的海潮波锋
  - 外放阶段会按距离从近到远依次命中敌方英雄；每个被外放波锋扫到的目标都会受到一次范围伤害并附加 `0.75 秒` `KnockUp`
  - 海潮到达最大半径后，会短暂停顿再向中心回卷；每个被回卷波锋扫到的目标都会再次受到一次较低的范围伤害并附加 `0.5 秒` `KnockUp`
  - 同一目标在外放阶段最多命中 `1` 次，在回卷阶段最多命中 `1` 次；如果目标在两个阶段都处于波锋路径上，就允许吃到两段命中
  - 外放单次伤害为基于攻击力 `4.0` 倍率
  - 回卷单次伤害为基于攻击力 `2.5` 倍率
  - 若标准脆皮在一次大招中外放与回卷都完整吃到，应被明显压进危险线；但这招的核心价值仍是双程打断和团战控场，不是由潮汐猎人单独完成击杀
  - 大招不默认附带水平击退、拉拽或吸附；“海浪掀起目标”的核心语义继续由统一 `KnockUp` 表达
- 检测条件：
  - 本局尚未释放过大招
  - 优先要求自身 `8.5` 范围内至少存在 `3` 名敌方英雄
  - 若其中至少 `2` 名敌方英雄位于 `5.5` 范围内，则显著提高释放优先级
  - 若开战 `45 秒` 后仍未满足理想窗口，则只要自身 `8.5` 范围内仍有至少 `2` 名敌方英雄即可释放
- 当前关键参数：
  - 最大作用半径：`8.5`
  - 外放阶段持续时间：`0.75 秒`
  - 外圈停顿：`0.2 秒`
  - 回卷阶段持续时间：`0.75 秒`
  - 波锋厚度：`1.4`
  - 外放伤害：基于攻击力 `4.0`
  - 回卷伤害：基于攻击力 `2.5`
  - 外放击飞持续时间：`0.75 秒`
  - 回卷击飞持续时间：`0.5 秒`
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 潮汐猎人当前定位为“第三个坦克扩展原型”，不替代 `tank_001_ironwall` 的稳团保护模板，也不替代 `tank_002_shieldwarden` 的单点与全体嘲讽模板
- `Undertow Carapace` 应优先复用共享的“最近直接受击来源窗口”运行时信息，不要为潮汐猎人单独维护一套仇恨表、攻击者列表或额外 `Update` 轮询
- 小技能的动态防御加成必须继续走统一属性汇总入口；来源变化只刷新正式防御加成，不直接改写伤害公式
- `Tidal Rebound` 应优先落成“记录施法中心点 + 按时间顺序结算多层环带命中 + 外放后反向回卷再结算一次”的共享技能表达，不要直接做成依赖物理碰撞体实时扫描的潮汐猎人专属圆环脚本
- 为保持第一阶段实现简单，回卷终点固定为施法瞬间记录的中心点，不实时跟随潮汐猎人后续移动；如果未来要改成跟随施法者，必须先补文档
- 外放与回卷的击飞都必须直接复用统一 `KnockUp` 状态语义；如需画面抬升、水浪拖尾或地面浪纹扩散，由表现层处理，不新增“海浪击飞”这类专属状态类型
- 日志和事件至少要能明确记录：当前有效承压来源数、被动防御加成刷新、大招开始、外放命中数、回卷命中数，以及各目标的击飞持续时间

## 设计边界

- 潮汐猎人的核心职责是前排承伤、反开和范围打断，不承担高爆发收割职责
- 不应把潮汐猎人做成高机动切后排或跨半场先手突进的主动开团坦克
- 不应把潮汐猎人做成兼具全队防御 Buff、群体嘲讽、分担伤害和双程击飞的全能主控核心
- 不应把 `Tidal Rebound` 扩成全屏覆盖、超长控制链或附带明显拉拽 / 吸附的超模团控技能
- 不应把 `Undertow Carapace` 扩成同时提供护盾、回血、减伤和反伤的全能被动；它当前只负责“被集火时动态增防”
- 第一阶段下，潮汐猎人优先验证“近战普攻 + 被集火实时增防小技能 + 往返扩散海潮大招”这套正面控场坦克模板
