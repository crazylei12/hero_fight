# Rifleman

## 基础信息

- 英文名：Rifleman
- 中文名：步枪手
- 英雄 ID：`marksman_002_rifleman`
- 英雄分类：射手
- 标签：远程、持续输出、单点压制、脆皮
- 核心定位：后排定点压制型射手，依靠高威胁的慢速步枪普攻持续点杀单个目标，用短时连射小技能快速压低一个敌人的血线，再用一局一次的手榴弹大招惩罚敌方抱团
- 默认 AI 模板：`marksman_default`
- 当前定位补充：作为 `marksman_001_longshot` 之后的第二个射手扩展原型，不替代迅捷射手的“拉扯 + 击退 + 连射收割”模板，主要验证“慢速高威胁投射物普攻 + 单体技能连射 + 纯伤害范围手榴弹大招”这条更笨重的射手分支

## 当前数值倾向

- 单发普攻威胁应高于 `Longshot`，但普攻频率明显更慢
- 生存性应低于当前大多数后排，体现“低防御、容易被贴脸击杀”的来源特征
- 移速应低于 `Longshot`，不依赖频繁拉扯或自保位移吃饭
- 射程可与 `Longshot` 持平或略长，但不能因为射程优势掩盖其机动性短板

## 当前实现基线

- 最大生命：`280`
- 攻击力：`40`
- 防御力：`6`
- 攻速：`0.699`，约等于 `1.43 秒 / 次`
- 移速：`3.2`
- 暴击率：`18%`
- 暴击伤害：`1.75`
- 攻击距离：`9.3`

## 普攻类型

- 类型：远程投射物普攻
- 释放目标：当前索敌目标；默认沿用射手模板维持后排输出位置，持续攻击最近的可攻击敌人，被近身压迫时仍可短时后撤，但不会像 `Longshot` 一样依赖额外击退技能重新拉开距离
- 效果：发射一发高威胁步枪弹，对命中的敌方目标造成基于攻击力 `1.15` 倍率的单体伤害
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`9.3`
  - 普攻间隔：`1.43 秒`
  - 投射物速度：`18`

## 小技能：Burst Fire

- 技能类型：连续单体伤害序列
- 释放目标：当前按统一 `NearestEnemy` 目标选择执行，直接选择施法距离内最近的合法敌方英雄
- 效果：
  - 短时锁定一个敌方目标，快速连续打出 `4` 发技能射击
  - 每发射击都造成基于攻击力 `0.55` 倍率的单体伤害
  - 连射期间步枪手不获得位移、护盾、无敌或不可选中
  - 若目标在连射过程中死亡、脱离合法性或进入不可直接选中的状态，则剩余射击立即取消，不自动转火
  - 若步枪手在连射过程中被眩晕、击飞或其他会打断施法的效果命中，剩余射击也会立即取消
- 检测条件：
  - 小技能 CD 转好
  - 施法距离 `9.3` 内存在合法敌方目标
- 当前关键参数：
  - 施法距离：`9.3`
  - 总射击次数：`4`
  - 连射间隔：`0.18 秒`
  - 单次射击伤害：基于攻击力 `0.55`
  - 冷却时间：`5 秒`

## 大招：Frag Grenade

- 技能类型：投掷范围伤害
- 释放目标：优先选择施法距离内敌人最密集的位置；若当前攻击目标附近已经形成可观聚集，也可直接朝该目标当前位置投掷
- 效果：
  - 通过共享投掷表现层，从施法者手部锚点朝目标点抛出一枚手榴弹
  - 手榴弹在短暂飞行后落地爆炸，对范围内所有敌方英雄造成一次基于攻击力 `7` 倍率的范围伤害
  - 满额命中满血脆皮时，应接近或直接击杀；中坚应被重伤；坦克仍应存活但不健康
  - 该大招当前只承担“范围爆发收口”职责，不默认附带眩晕、击飞、减速、持续灼烧或击退
  - 落点在投出时确定；若原目标随后死亡或失去合法性，手榴弹仍会在既定目标点爆炸
- 检测条件：
  - 本局尚未释放过大招
  - 优先要求施法距离 `10.5` 内的目标点半径 `6` 内至少存在 `3` 名敌方英雄
  - 若开战 `40 秒` 后仍未满足，则阈值降低为至少 `2` 名敌方英雄
  - 若开战 `52 秒` 后仍未满足，则只要施法距离内存在合法敌方目标即可释放
- 当前关键参数：
  - 施法距离：`10.5`
  - 作用半径：`6`
  - 爆炸伤害：基于攻击力 `7`
  - 投掷飞行时间：`0.45 秒`
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 步枪手当前定位是“第二个射手扩展原型”，不替代 `Longshot` 作为首个拉扯型射手模板
- 普攻的差异重点应首先落在“更高单发伤害、更慢攻击频率、更差生存性”上，不应再额外补一套强自保机制把职业差异抹平
- `Burst Fire` 应优先复用统一的“连续动作序列”能力，但这次的动作载荷应是“重复执行同一个技能伤害段”，而不是 `Longshot` 大招那种“重复执行普攻”
- `Burst Fire` 当前已经按共享 `NearestEnemy` 目标选择接入；如果后续真要补“优先当前攻击目标”的语义，应先补统一目标选择能力，再改文档
- `Burst Fire` 的目标失效处理应作为共享序列配置能力表达：
  - 允许声明“目标失效后立即取消剩余序列”
  - 而不是默认所有连续序列都自动刷新到新目标
- `Frag Grenade` 应优先复用统一的投射物或延迟命中范围技能结构，不要为这名英雄单独写“手榴弹专属结算底层”
- `Frag Grenade` 的爆炸属于范围效果；即使爆炸范围内有处于 `不可选中` 状态的敌人，只要统一范围命中规则认定这类范围效果仍可命中，就应继续正常结算，不要把它误做成“只能炸中投掷时锁定的那个目标”
- 如果后续技能模板暂时不足以表达“技能序列命中取消策略”或“带飞行时间的落点爆炸”，应优先补共享配置和共享执行入口，不要把步枪手写成孤立的射手特例

## 设计边界

- 步枪手的核心职责是后排单点压制和补足一段范围爆发，不承担前排承伤或持续控制职责
- 不应把步枪手做成拥有高机动位移、稳定保命或长时间不可选中的远程刺客
- 不应把手榴弹大招扩成法师式持续区域压制或大范围硬控技能
- 不应把 `Burst Fire` 扩成可以无脑自动转火、无视控制打满全部射击的强保底技能
- 第一阶段下，步枪手优先验证“慢速高伤普攻 + 单体短连射技能 + 一局一次投掷型范围爆发大招”这套笨重火力型射手模板
