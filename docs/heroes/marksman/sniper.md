# Sniper

## 基础信息

- 英文名：Sniper
- 中文名：狙击手
- 英雄 ID：`marksman_005_sniper`
- 英雄分类：射手
- 标签：远程、超长射程、后排威胁、单体爆发、瞄准序列
- 核心定位：后排定点狙杀型射手，用全射手最长的基础攻击距离保持安全输出，但基础攻速明显更低；普通状态下仍会优先攻击最近可攻击敌人，真正的差异点来自小技能和大招能够绕过前排，稳定威胁敌方后排或最远目标
- 默认 AI 模板：`marksman_default`
- 当前定位补充：作为 `marksman_004_boomeranger` 之后的第五个射手扩展原型，不替代 `Longshot` 的连射收割、不替代 `Rifleman` 的慢枪前排压制、不替代 `Venomshooter` 的 DOT 收割，也不替代 `Boomeranger` 的短射程 AOE；狙击手主要验证“最长基础射程 + 指定范围内最远敌人目标策略 + 多目标唯一狙击序列”这条反后排射手方向

## 当前数值倾向

- 基础攻击距离应是当前射手中最长，明显长于 `Rifleman`，但不等同于全屏普攻
- 基础攻速应是当前射手中最低之一，让狙击手不能只靠普通攻击压过所有前排
- 攻击力保持中等偏上，但不做成每发普攻都接近小技能的重炮
- 生存性应偏低；狙击手的反制点是被刺客、钩子、换位或强制贴脸打乱瞄准节奏
- 移速不应过高，避免最长射程叠加高机动后变成没有明显反制窗口的安全输出位

## 当前实现基线

- 最大生命：`275`
- 攻击力：`42`
- 防御力：`5`
- 攻速：`0.556`，约等于 `1.80 秒 / 次`
- 移速：`3.1`
- 暴击率：`20%`
- 暴击伤害：`1.85`
- 攻击距离：`12`

## 普攻类型

- 类型：远程投射物普攻
- 释放目标：当前索敌目标；默认沿用射手模板维持后排输出位置，仍按当前共享敌方普攻目标规则攻击最近的可攻击敌人，不通过普攻额外锁定敌方后排
- 效果：发射一发精准子弹，对命中的敌方目标造成基于攻击力 `1.0` 倍率的单体伤害
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`12`
  - 普攻间隔：`1.80 秒`
  - 投射物速度：`24`

## 小技能：Deadeye Shot

- 技能类型：超距单体伤害 / 最远敌人点名
- 释放目标：以施法者当前位置为中心，在 `基础攻击距离 x 2` 范围内选择当前距离施法者最远的合法敌方英雄；若距离并列，则优先当前绝对生命值更低者，再比较当前生命值比例
- 效果：
  - 狙击手短暂瞄准后，对目标进行一次强力狙击
  - 这次命中是直接指定目标的单体技能伤害，不是会被前排单位拦截的直线碰撞弹道
  - 对目标造成基于攻击力 `2.35` 倍率的单体伤害
  - 该技能的核心职责是让射手阵容具备稳定触达后排的能力，而不是继续把全部伤害灌到最近前排身上
  - 技能不附带眩晕、击退、减速、破甲或斩杀，避免把反后排能力同时扩成强控制或全能收割
- 检测条件：
  - 小技能 CD 转好
  - `基础攻击距离 x 2` 范围内存在合法敌方英雄
  - 狙击手没有被眩晕、击飞或其他禁止主动施法的效果限制
- 当前关键参数：
  - 施法距离：`基础攻击距离 x 2`，按当前攻击距离 `12` 计算为 `24`
  - 伤害倍率：基于攻击力 `2.35`
  - 瞄准前摇：沿用当前主动技能基础前摇
  - 冷却时间：`7 秒`

## 大招：Kill Zone

- 技能类型：主动型大招 / 瞄准状态 / 多目标唯一狙击序列
- 释放目标：大招开始时，收集当前所有合法敌方英雄，并按“距离狙击手从远到近”排序；每名敌方英雄在本次大招中最多被狙击 `1` 次
- 效果：
  - 狙击手进入短暂瞄准状态，停止移动和普攻，按排序结果依次对敌方英雄各狙击一次
  - 每次狙击造成基于攻击力 `3.05` 倍率的单体伤害
  - 排序优先从最远目标开始，确保大招即使被中途打断，也会先把价值集中到敌方后排或远端单位身上
  - 每次狙击在开枪前都会重新确认该目标仍然存活、可被直接指定且仍是合法敌方英雄；若目标已经死亡、复活等待、不可选中或失去合法性，则跳过该目标并继续下一个目标
  - 大招开始后新复活或新进入合法状态的敌人，不加入本次已快照的目标列表，避免一局一次大招在复活规则下无限延伸
  - 大招不附带位移、无敌、不可选中或霸体；若狙击手在瞄准状态中被眩晕、击飞或其他会打断施法的效果命中，剩余狙击立即取消
  - 该大招按“覆盖多名敌人的纯伤害序列”理解：它应明显压低敌方后排血线并制造收割窗口，但不应稳定从满血状态直接蒸发整个敌队
- 检测条件：
  - 本局尚未释放过大招
  - 当前至少存在 `3` 名合法敌方英雄
  - 当前实现基线优先在敌方仍有至少 `4` 名合法敌方英雄时释放
  - 若开战 `38 秒` 后仍未满足，则只要当前至少存在 `3` 名合法敌方英雄即可释放
  - 若开战 `52 秒` 后仍未释放，则只要当前至少存在 `1` 名合法敌方英雄即可释放，避免一局结束前完全空过
- 当前关键参数：
  - 单个目标最大命中次数：`1`
  - 理想狙击目标数：`5`
  - 狙击间隔：`0.35 秒`
  - 单次狙击伤害：基于攻击力 `3.05`
  - 大招期间临时施法射程：`全屏`，当前 `arena_stage01_flat` 可按不低于 `56` 处理
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 狙击手的普攻仍应遵守共享敌方普攻索敌规则；不要为了突出反后排，把普攻也私下改成优先后排或最远敌人
- `Deadeye Shot` 应优先复用共享的 `FarthestEnemyFromSelf` 或等价目标策略，过滤条件是“施法者指定范围内合法敌方英雄”，排序条件是“距离施法者最远”
- `Deadeye Shot` 的施法距离应从基础攻击距离派生，当前写成 `基础攻击距离 x 2`；后续如果调整狙击手基础射程，应同步核对小技能实际施法距离
- 小技能和大招的狙击命中都应走统一单体技能伤害、暴击 / 非暴击规则、目标合法性复检、事件和日志入口，不要直接在单英雄脚本里扣血
- 大招应优先落到共享“连续动作序列”能力的一个扩展实例：目标列表按规则快照、每个目标最多执行一次、目标失效时跳过、被硬控时中断
- 如果现有连续动作序列只能表达“反复打同一个动作直到次数耗尽”，后续实现应扩共享序列目标列表能力，而不是给 `marksman_005_sniper` 写一套私有大招循环
- 事件与日志至少要能明确记录：小技能本次选中了哪个最远目标、大招开始时快照了几个目标、大招每一枪命中了谁、哪些目标因失效被跳过，以及大招是否被控制中断

## 设计边界

- 狙击手的核心职责是超长射程安全输出和反后排点名，不承担前排承伤、团队保护或大范围控场职责
- 不应把狙击手做成普攻也稳定无视前排的全自动后排处决机器；它的后排威胁应主要来自小技能和一局一次大招窗口
- 不应给狙击手补位移、隐身、无敌、不可选中或强击退来自保；被刺客和搅阵英雄打乱瞄准，是这名英雄应有的反制点
- 不应把 `Deadeye Shot` 扩成斩杀、穿透一条线或附带硬控的万能技能
- 不应把 `Kill Zone` 做成稳定全队满血秒杀；它应制造后排血线崩盘和团队收割窗口，但需要队友跟进或前置压血才能最大化收益
- 第一阶段下，狙击手优先验证“最长射程慢速普攻 + 指定范围最远敌人小技能 + 一局一次全队唯一狙击序列”这套反后排射手模板
