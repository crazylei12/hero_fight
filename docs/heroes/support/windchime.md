# Windchime

## 基础信息

- 英文名：Windchime
- 中文名：风语司铃
- 英雄 ID：`support_002_windchime`
- 英雄分类：辅助
- 标签：远程、保护、减益、反切入、控制
- 核心定位：中后排反切入保护辅助，依靠远程普攻持续削弱突进者输出，用单体保护技能帮己方后排拉开身位，再用一局一次的静风领域拆对方开团节奏
- 默认 AI 模板：`support_default`
- 当前定位补充：作为 `support_001_sunpriest` 之后的第二个辅助扩展原型，不替代圣职者的“治疗续航 + 单点抬血”模板，主要验证“普攻 on-hit 减攻 + 单体反切入保护 + 反开团区域压制”这条保护控制型辅助方向

## 普攻类型

- 类型：远程投射物普攻
- 释放目标：默认站在己方中后排附近；若有敌方近战、刺客或坦克压进任一友方远程英雄 `4.5` 范围内，则优先攻击其中距离该友军最近者；否则退回当前最近的合法敌方英雄
- 效果：
  - 发射风铃弹，对命中的敌方目标造成基于攻击力 `1.0` 倍率的单体伤害
  - 命中后为目标附加一次 `AttackPowerModifier`
  - 单层减攻幅度为 `-6%`
  - 单次持续时间为 `3 秒`
  - 同目标可叠加至 `3 层`
  - 重复命中时刷新持续时间
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`8.4`
  - 普攻间隔：`1.05 秒`
  - 投射物速度：`14`

## 小技能：Echo Canopy（回音庇幕）

- 技能类型：单体保护 / 反切入保护
- 释放目标：优先选择施法距离 `9.2` 内当前 `3.5` 范围内已有敌方近战、刺客或坦克压近的友方远程英雄；若有多个，则优先当前生命值比例更低者；若当前没有被近身威胁的远程友军，则退回施法距离内生命值最低的友方远程英雄
- 效果：
  - 立即为目标施加一个护盾
  - 同时附加一次 `MoveSpeedModifier`
  - 移速提升幅度为 `+35%`
  - 移速增益持续 `2.5 秒`
  - 额外附加一层持续 `3 秒` 的“庇幕保护”
  - 庇幕保护期间，若有敌方英雄进入目标 `2.4` 范围内，则立刻以目标为中心触发一次风震
  - 风震会对范围内所有敌方英雄附加 `0.6 秒` `KnockUp`
  - 若风震成功击飞敌方英雄，则每命中 `1` 名，都会为受保护目标回复 `15` 点生命值
  - 同时沿远离受保护友军的方向施加一次水平击退
  - 击退距离为 `1.8`
  - 这次反击只会触发 `1 次`，触发后本次庇幕保护立即消耗
  - 技能核心收益是“敌人即使贴到后排，也会被当场打断并推开一点”，为己方射手或法师争取重新拉开距离的时间
- 检测条件：
  - 小技能 CD 转好
  - 施法距离 `9.2` 内存在合法友方目标
- 当前关键参数：
  - 施法距离：`9.2`
  - 护盾值：`55`
  - 移速提升：`+35%`
  - 移速增益持续时间：`2.5 秒`
  - 庇幕保护持续时间：`3 秒`
  - 近身触发半径：`2.4`
  - 击飞持续时间：`0.6 秒`
  - 每次成功击飞回复：`15`
  - 击退距离：`1.8`
  - 冷却时间：`8 秒`

## 大招：Stillwind Domain（静风禁域）

- 技能类型：范围控制 / 区域压制 / 团战反开
- 释放目标：若任一友方远程英雄 `4.5` 范围内已有至少 `2` 名敌方英雄压进，则优先以该友军当前位置为中心释放；若当前没有受压后排目标，则在施法距离 `10.5` 内选择敌方最密集区域释放
- 效果：
  - 在目标点生成一个固定不跟随的“静风领域”
  - 领域生成瞬间，对区域内所有敌方英雄附加 `0.75 秒` `KnockUp`
  - 同时为区域内所有友方英雄施加一次护盾
  - 领域持续期间，每 `1 秒` 对区域内敌方英雄附加或刷新一次 `MoveSpeedModifier`
  - 减速幅度为 `-30%`
  - 同时附加或刷新一次 `AttackPowerModifier`
  - 减攻幅度为 `-20%`
  - 两种减益的单次持续时间都为 `1.2 秒`
  - 领域在释放后固定于原地，不跟随施法者或友军移动
  - 大招核心收益是反开团和团战拆节奏，不依赖直接伤害收割，而是先把冲进来的敌人打断，再让他们在领域里变慢、降伤害，为己方后排争取安全输出时间
- 检测条件：
  - 本局尚未释放过大招
  - 优先在“友方远程英雄被至少 `2` 名敌人压近”的情况下释放
  - 若当前没有明显后排受压点，则要求施法距离 `10.5` 内能找到半径 `6.5` 范围内至少存在 `3` 名敌方英雄的区域锚点
  - 若开战 `45 秒` 后仍未满足上述条件，则阈值降低为至少 `2` 名敌方英雄
- 当前关键参数：
  - 施法距离：`10.5`
  - 区域半径：`6.5`
  - 区域持续时间：`5 秒`
  - 脉冲间隔：`1 秒`
  - 敌方击飞持续时间：`0.75 秒`
  - 敌方减速幅度：`-30%`
  - 敌方减攻幅度：`-20%`
  - 敌方减益单次持续时间：`1.2 秒`
  - 友方护盾值：`90`
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 风语司铃当前定位为“第二个辅助扩展原型”，不替代 `support_001_sunpriest` 作为首个治疗续航辅助模板
- 普攻附带减攻必须直接复用统一的“普攻命中附带效果”配置能力，不应偷偷实现成风语司铃专属被动脚本
- 普攻与小技能的目标选择，都应优先沉淀为共享的“优先保护被近身威胁的友方远程单位”过滤 + 排序 + 回退策略，不要只在风语司铃流程里单独硬编码
- `Echo Canopy` 的风震反击应优先表达为“附着在友军身上的短时保护效果 + 单次近身触发”，或等价的跟随友军的一次性警戒区；如果现有技能模板不足，应先补共享能力，不要把持续近身检测散写进风语司铃专属 AI 或表现脚本
- 风震反击的打断效果必须复用 `KnockUp + ApplyForcedMovement` 的组合语义，不新增“吹飞”或“弹开”这类新的专属状态类型
- `Stillwind Domain` 应优先复用统一的“持续区域 + 周期脉冲载荷”结构：生成瞬间处理一次敌方击飞和友方护盾，后续脉冲只负责刷新减速与减攻，不要额外创建“全场进入静风模式”的专属战斗状态
- 大招虽然是区域压制技能，但当前不应额外附带持续伤害、治疗、无敌或不可选中，避免把这名辅助做成功能过满的全能团战核心

## 设计边界

- 风语司铃的核心职责是保护己方后排和拆解敌方切入，不承担主治疗续航职责
- 不应把风语司铃做成靠高频护盾和持续治疗维持全队血线的第二个圣职者
- 不应把风语司铃做成高爆发法师式辅助，或依赖大范围伤害收割残局的输出位
- 不应把风语司铃的大招扩成长时间硬控链或超大范围封场技能；它更应该是一段短而强的反开窗口
- 第一阶段下，风语司铃优先验证“远程投射物普攻减攻 + 单体反切入保护 + 一局一次区域反开大招”这套保护控制型辅助模板
