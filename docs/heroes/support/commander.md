# Commander

## 基础信息

- 英文名：Commander
- 中文名：指挥官
- 英雄 ID：`support_006_commander`
- 英雄分类：辅助
- 标签：远程、增益、集火、减防、团队输出
- 核心定位：中后排战术增伤辅助，稳定把攻击力增益给到当前己方主输出，再用一局一次的集火指令压低敌方关键单位防御并引导全队转火
- 默认 AI 模板：`support_default`
- 当前定位补充：作为 `support_001_sunpriest`、`support_002_windchime`、`support_003_monk`、`support_004_shrinemaiden` 和 `support_005_chef` 之后的第六个辅助扩展原型，不替代已有治疗、护盾、反切入或持续供给辅助；主要验证“稳定定向输出增益 + 团队集火标记 + 减防”的纯输出放大辅助方向

## 普攻类型

- 类型：远程投射物普攻
- 释放目标：默认保持在己方中后排附近，按共享 `NearestEnemy` 目标策略攻击最近的合法敌方英雄；若存在当前团队集火标记目标，且该目标仍为合法敌方目标，则优先攻击集火标记目标
- 效果：
  - 发射一枚指挥弹，对命中的敌方目标造成基于攻击力 `0.9` 倍率的单体伤害
  - 普攻本身不附带治疗、护盾、控制或额外减益；它的职责是让指挥官保持基础参战节奏，而不是成为远程输出核心
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`8.5`
  - 普攻间隔：`1.12 秒`
  - 投射物速度：`14`

## 小技能：Battle Orders（火力号令）

- 技能类型：单体攻击力增益
- 释放目标：
  - 在施法距离 `10` 内选择当前对局内累计造成伤害最高的合法友方英雄，包括自己
  - 若累计造成伤害并列，则优先当前攻击力最终值更高者；若仍并列，则优先当前生命值比例更高者；若仍并列，则优先更近者
  - 这条排序的目标是稳定把增益给到当前主输出，而不是在低血量队友身上浪费成保护技能
- 效果：
  - 为目标附加一次 `AttackPowerModifier`
  - 攻击力提升幅度为 `+25%`
  - 增益持续 `5 秒`
  - 同一名指挥官对同一目标的这条增益默认 `maxStacks = 1`，重复施加只刷新持续时间
  - 这招不附带治疗、护盾、减伤或攻速提升；稳定性来自“明确找当前最高输出友军”，而不是一次给多个属性
- 检测条件：
  - 小技能 CD 转好
  - 施法距离 `10` 内存在合法友方目标
- 当前关键参数：
  - 施法距离：`10`
  - 攻击力提升：`+25%`
  - 增益持续时间：`5 秒`
  - 冷却时间：`7 秒`

## 大招：Focus Fire Command（集火号令）

- 技能类型：持续团队集火指令 + 单体减防，可在持续时间内迁移目标
- 初始释放目标：
  - 在施法距离 `11` 内选择当前对局内累计造成伤害最高的合法敌方英雄
  - 若累计造成伤害并列，则优先当前绝对生命值更低者；若仍并列，则优先距离指挥官更近者
- 效果：
  - 创建一个持续 `6 秒` 的团队集火指令
  - 当前指令目标会获得 `集火标记 / FocusFireMark`
  - 指挥官同阵营的英雄在进行敌方伤害普攻目标选择时，应优先选择这个集火标记目标
  - 若某个敌方单体伤害技能显式声明可接受团队集火覆盖，且集火标记目标仍为合法目标，也应优先选择这个目标
  - 友方治疗、护盾、增益、保护类技能不受集火标记影响
  - 当前指令目标同时获得一次 `DefenseModifier`
  - 防御力下降幅度为 `-35%`
  - 减防持续时间跟随该目标当前持有的集火标记，不额外独立延长
  - 若当前指令目标死亡、进入复活等待、变为不可选中或不再是合法敌方单位，而大招剩余持续时间仍未结束，则立刻按同一目标策略重新寻找下一个合法敌方英雄，并把剩余持续时间的集火标记和减防转移到新目标身上
  - 如果短时间内没有合法敌方目标，指令不延长持续时间；只在剩余时间内继续等待合法目标重新出现
  - 施放成功后，指令作为本次大招的运行时对象存在；即使指挥官之后死亡或等待复活，已经发出的集火指令也不提前结束，日志归属仍挂回指挥官和本次大招
  - 这招不直接造成伤害，不附带眩晕、击飞、沉默或现有 `Taunt`，核心收益来自“全队更少分散火力 + 目标更脆”
- 检测条件：
  - 本局尚未释放过大招
  - 施法距离 `11` 内存在合法敌方英雄
  - 优先要求我方当前至少 `3` 名英雄存活，且至少 `2` 名不是死亡、复活等待或硬控状态，避免在无人能跟进时浪费大招
  - 若开战 `40 秒` 后仍未释放，则门槛放宽为我方至少 `2` 名英雄存活且存在合法敌方目标
  - 若进入加时仍未释放，只要存在合法敌方目标即可释放
- 当前关键参数：
  - 施法距离：`11`
  - 指令持续时间：`6 秒`
  - 防御力下降：`-35%`
  - 集火目标迁移：目标失效后按剩余持续时间迁移，不刷新总持续时间
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 指挥官当前定位为“稳定输出放大型辅助”，不替代 `support_005_chef` 的随机供给和团队续航，也不替代已有保护型辅助
- `Battle Orders` 必须复用正式 `AttackPowerModifier` 状态语义，不要把攻击力提升散写进普攻伤害公式或技能倍率特判
- `Battle Orders` 所依赖的“当前累计造成伤害最高的合法友方英雄”应统一读取 `RuntimeHero` 或等价运行时统计，不要让指挥官自己维护一份私有输出排行榜
- `Focus Fire Command` 不能复用现有 `Taunt` 状态：现有 `Taunt` 的语义是让被嘲讽者攻击状态来源并禁止其主动施法，而本大招需要的是让指挥官阵营优先攻击被标记者
- `Focus Fire Command` 应优先落成共享 `FocusFireMark` 或等价的统一团队目标规则，不要在每个友军 AI 里分别硬编码“如果队友是指挥官就改目标”
- 集火指令的迁移逻辑应由本次大招的运行时指令对象统一管理；目标死亡后换到谁、剩余多少时间、减防是否同步迁移，都必须能被日志追踪
- 集火标记只应影响敌方伤害目标选择，不应强制友军停止治疗、护盾或保护技能，也不应让友军无视眩晕、击飞、不可选中、死亡和复活等待等正式行为门禁
- 同一阵营同一时间默认只保留一个团队集火目标；若未来允许多个指挥官同场，新的集火指令默认覆盖旧指令，避免同一队伍出现多个强制优先目标
- 日志和调试事件至少应能明确记录：指挥官释放大招、初始标记目标、每次目标迁移原因、当前剩余持续时间、减防开始与结束

## 设计边界

- 指挥官的核心职责是放大队友输出，不承担主治疗、主护盾或反切入保护职责
- 不应把指挥官做成高额持续输出位；它的普攻只提供基础参战，不应接近射手或法师的主输出能力
- 不应把 `Battle Orders` 扩成同时给攻击力、攻速、暴击率和暴击伤害的全套输出爆发包；第一阶段只稳定提供攻击力增益
- 不应把 `Focus Fire Command` 写成全场永久锁头、完整仇恨系统或能同时标记多个敌人的复杂指挥体系
- 不应把 `Focus Fire Command` 误做成现有 `Taunt`、沉默、眩晕或缴械；被标记敌人仍能移动、普攻和主动施法
- 不应让集火标记覆盖所有技能目标；治疗、保护和非敌方伤害技能仍应按自己的目标策略运行
- 第一阶段下，指挥官优先验证“远程投射物普攻 + 稳定单体攻击力增益小技能 + 可迁移团队集火减防大招”这套输出放大型辅助模板
