# Chef

## 基础信息

- 英文名：Chef
- 中文名：厨子
- 英雄 ID：`support_005_chef`
- 英雄分类：辅助
- 标签：远程、治疗、增益、持续回复
- 核心定位：中后排供给型持续回复辅助，依靠友方投射物普攻把轻量 `披萨回复` 稳定铺到队友身上，用短 CD 的随机快餐定向抬高核心输出或前排承伤，再用一局一次的团队盛宴把全队从持续换血线拉回可继续作战的健康线
- 默认 AI 模板：`support_default`
- 当前定位补充：作为 `support_001_sunpriest`、`support_002_windchime`、`support_003_monk` 和 `support_004_shrinemaiden` 之后的第五个辅助扩展原型，不替代圣职者的单点急救模板，不替代风语司铃的反切入保护模板，不替代僧侣的贴前排群体续航模板，也不替代巫女的双态节奏模板；主要验证“友方投射物普攻铺持续治疗 + 按对局内统计定向投喂 Buff + 一局一次全队即时治疗与持续回复大招”这条纯团队供给型辅助方向

## 普攻类型

- 类型：远程投射物治疗普攻 / 优先补齐团队持续回复覆盖
- 释放目标：
  - 优先选择攻击距离 `8.4` 内当前没有 `PizzaRegen / 披萨回复` 的合法友方英雄，包括自己
  - 若有多个无披萨目标，则优先当前绝对生命值更低者；若仍并列则优先更近者
  - 若当前所有合法友军都已有 `披萨回复`，则回退到该效果剩余持续时间最短的友军；若仍并列则优先当前绝对生命值更低者，再并列选更近者
  - 当前没有受伤友军时仍可正常投喂，用于维持团队覆盖，不应原地空等
- 效果：
  - 发射一块披萨，对命中的友军附加一次 `HealOverTime`
  - `披萨回复` 持续 `4 秒`
  - 每 `1 秒` 回复一次生命值
  - 每跳治疗量为基于攻击力 `0.25` 倍率
  - 同一名厨子对同一目标的这条 `披萨回复` 默认 `maxStacks = 1`，重复命中只刷新持续时间
  - 普攻本体不附带直接治疗、伤害或额外属性变化；它的核心收益是把轻量持续回复尽可能铺给更多友军
- 检测条件：
  - 当前存在合法友方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`8.4`
  - 普攻间隔：`1.05 秒`
  - 投射物速度：`13.5`
  - 披萨回复持续时间：`4 秒`
  - 披萨回复跳数：`4`
  - 单跳治疗倍率：基于攻击力 `0.25`

## 小技能：Daily Special

- 技能类型：单体增益投射 / 施放时随机决定菜品
- 释放目标：
  - 每次释放时，先用统一随机数服务在 `汉堡`、`烤肠`、`薯条` 三种菜品中等概率选出本次结果
  - 若本次为 `汉堡`，则在施法距离 `10` 内选择当前累计造成伤害最高的合法友方英雄；若并列则优先当前生命值比例更低者，若仍并列则优先更近者
  - 若本次为 `烤肠`，则在施法距离 `10` 内选择当前累计承受伤害最高的合法友方英雄；若并列则优先当前生命值比例更低者，若仍并列则优先更近者
  - 若本次为 `薯条`，则在施法距离 `10` 内选择当前累计造成伤害最高的合法友方英雄；若并列则优先当前生命值比例更低者，若仍并列则优先更近者
- 效果：
  - 发射本次随机到的食物投射物，并在命中后附加对应单体增益
  - `汉堡`：附加一次 `AttackPowerModifier`
  - 攻击力提升幅度为 `+20%`
  - `烤肠`：附加一次 `DefenseModifier`
  - 防御力提升幅度为 `+25%`
  - `薯条`：附加一次 `AttackSpeedModifier`
  - 攻速提升幅度为 `+20%`
  - 三种增益的单次持续时间都为 `4.5 秒`
  - 这招不附带直接治疗或伤害，核心收益是用较短 CD 持续把正确属性喂给当前最需要的队友
- 检测条件：
  - 小技能 CD 转好
  - 施法距离 `10` 内存在合法友方目标
- 当前关键参数：
  - 施法距离：`10`
  - 攻击力提升：`+20%`
  - 防御力提升：`+25%`
  - 攻速提升：`+20%`
  - 增益持续时间：`4.5 秒`
  - 冷却时间：`5.5 秒`

## 大招：Grand Feast

- 技能类型：全队即时治疗 + 持续回复 + 攻防增益
- 释放目标：自己；施法时对全场所有当前存活的友方英雄生效，包括自己
- 效果：
  - 立即向每名当前存活友军扔出一个 `巨型汉堡`
  - 命中后先造成一次基于攻击力 `1.0` 倍率的直接治疗
  - 同时附加一次更强的 `HealOverTime`
  - 这条持续回复持续 `5 秒`
  - 每 `1 秒` 回复一次生命值
  - 每跳治疗量为基于攻击力 `0.30` 倍率
  - 额外为每名命中友军附加一次 `AttackPowerModifier`
  - 攻击力提升幅度为 `+15%`
  - 再附加一次 `DefenseModifier`
  - 防御力提升幅度为 `+15%`
  - 两种属性增益都持续 `5 秒`
  - 大招的核心收益是同时止住团队持续掉血，并制造一段全队攻防都更适合继续换血和追击的窗口；因为它覆盖全队且同时带双属性增益，raw 治疗不应按纯治疗大招的极限档位设计
- 检测条件：
  - 本局尚未释放过大招
  - 优先要求当前至少 `3` 名友军仍存活
  - 且其中至少 `2` 名友军生命值比例不高于 `65%`
  - 若开战 `45 秒` 后仍未满足，则门槛放宽为当前至少 `2` 名友军存活，且其中至少 `1` 名友军生命值比例不高于 `80%`
- 当前关键参数：
  - 生效目标：所有当前存活友军
  - 直接治疗倍率：基于攻击力 `1.0`
  - 持续回复持续时间：`5 秒`
  - 持续回复间隔：`1 秒`
  - 持续回复单跳倍率：基于攻击力 `0.30`
  - 攻击力提升：`+15%`
  - 防御力提升：`+15%`
  - 增益持续时间：`5 秒`
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 厨子当前定位为“第五个辅助扩展原型”，不替代 `support_001_sunpriest` 的单点保人模板，也不替代 `support_002_windchime` 的反切入保护模板，不替代 `support_003_monk` 的近战前线续航模板，更不替代 `support_004_shrinemaiden` 的双态节奏模板
- 普攻的 `披萨回复` 必须直接复用统一 `HealOverTime` 状态语义，不要额外创建“厨子专属回复池”或常驻光环系统
- 普攻的“优先无披萨目标 -> 再按剩余持续时间最短回退”的目标规则，应沉淀为共享的过滤、排序和回退组合，不要把“谁身上还有披萨”这套判断散写成厨子专属索敌脚本
- `Daily Special` 的随机菜品必须在施法开始时就完成判定，并把结果作为显式 `variantKey` 或等价字段继续传给投射物、命中结算和日志；不要让表现层在食物已经飞出去后再倒推它到底是哪种菜
- `Daily Special` 所依赖的“当前累计造成伤害最高的合法友方英雄”与“当前累计承受伤害最高的合法友方英雄”，应优先沉淀为共享友方目标策略或统一 helper，不要让厨子自己额外维护一份私有主 C / 主坦榜单
- `Grand Feast` 必须对每名当前存活友军分别走统一的直接治疗、持续治疗与属性 Buff 入口，不要把它实现成只要站在厨子附近就持续生效的团队光环，也不要创建“全队共享盛宴池”这类专属底层
- 日志与调试事件至少应能明确记录：普攻给谁补上或刷新了 `披萨回复`、小技能本次随机到了什么菜品、最终命中了谁，以及大招实际命中了多少友军

## 设计边界

- 厨子的核心职责是平滑全队换血和维持持续作战能力，不承担圣职者那种强单点急救职责
- 不应把厨子做成能靠普攻同时打敌人又奶队友的双修输出辅助；它当前普攻就是纯友方投喂
- 不应把厨子做成带强控、反切入打断或大范围减益的全能辅助；控制和拆阵不是这名英雄的重点
- 不应把 `Daily Special` 扩成一次同时扔出三种食物，或稳定指定食物无限循环；“随机菜品 + 定向投喂”本身就是这招的辨识点
- 不应把 `Grand Feast` 继续叠加护盾、无敌、复活或攻速 Buff，避免它在同一个大招里把所有保护语义都吃满
- 第一阶段下，厨子优先验证“友方投射物普攻铺持续治疗 + 按对局内统计投单体增益 + 一局一次全队即时治疗与持续回复大招”这套供给型辅助模板
