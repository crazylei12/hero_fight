# Monk

## 基础信息

- 英文名：Monk
- 中文名：僧侣
- 英雄 ID：`support_003_monk`
- 英雄分类：辅助
- 标签：近战、治疗、护盾、承伤
- 核心定位：前排近战续航辅助，主动混在己方近战英雄群中作战，依靠贴身范围治疗稳住前线血线，再用一局一次的大范围护盾顶住关键换血窗口
- 默认 AI 模板：`support_default`
- 当前定位补充：作为 `support_001_sunpriest` 和 `support_002_windchime` 之后的第三个辅助扩展原型，不替代前两者的后排保人模板，主要验证“近战普攻 + 贴身群体治疗 + 以自身为圆心的大范围护盾”这条副坦型辅助方向

## 普攻类型

- 类型：近战瞬时命中普攻
- 释放目标：当前索敌目标；这名辅助默认应更积极贴近己方前排或当前交战中的近战友军，进入接敌后优先攻击最近敌方英雄，避免长期滞留在后排
- 效果：造成基于攻击力 `0.95` 倍率的单体伤害
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`1.9`
  - 普攻间隔：`1.05 秒`

## 小技能：Renewing Pulse

- 技能类型：以自身为圆心的范围治疗
- 释放目标：以自己为施法中心；施法时收集自身 `4.5` 范围内的所有友方英雄，包括自己
- 效果：
  - 立即为范围内所有友军回复生命值
  - 每个目标的治疗量为基于攻击力 `0.9` 倍率的直接治疗
  - 技能收益强依赖抱团站位，更适合在前排持续换血时稳定抬住己方近战群，而不是远距离点救后排
- 检测条件：
  - 小技能 CD 转好
- 当前关键参数：
  - 施法半径：`4.5`
  - 直接治疗倍率：基于攻击力 `0.9`
  - 冷却时间：`8 秒`

## 大招：Guardian Mantra

- 技能类型：以自身为圆心的大范围群体护盾
- 释放目标：以自己为施法中心；施法时收集自身 `6.8` 范围内的所有友方英雄，包括自己
- 效果：
  - 立即为范围内所有友军附加一层护盾
  - 护盾值为 `85`
  - 护盾持续 `5 秒`
  - 护盾直接附着在友军单位身上，施法后会随各自单位继续存在，不要求友军始终留在僧侣身边
  - 大招的核心收益是为己方近战群和跟进输出位一起制造一段“可以继续站住换血”的团队承压窗口，不附带控制、伤害或无敌
- 检测条件：
  - 本局尚未释放过大招
  - 优先要求自身 `6.8` 范围内至少存在 `2` 名友方英雄
  - 且其中至少满足以下任一条件：
    - 至少 `2` 名友军生命值比例不高于 `70%`
  - 若开战 `45 秒` 后仍未满足，则主条件门槛放宽为自身 `6.8` 范围内至少存在 `2` 名友方英雄，且生命值阈值放宽到 `80%`
- 当前关键参数：
  - 施法半径：`6.8`
  - 护盾值：`85`
  - 护盾持续时间：`5 秒`
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 僧侣当前定位为“第三个辅助扩展原型”，不替代 `support_001_sunpriest` 的后排续航模板，也不替代 `support_002_windchime` 的反切入保护模板
- `Renewing Pulse` 和 `Guardian Mantra` 都应优先复用统一的“以施法者为圆心收集友军”的共享范围施法能力，不要为这名英雄单独写一套常驻治疗光环或常驻护盾光环系统
- 小技能的范围治疗应继续逐个目标走统一 `HealApplied` 结算与日志事件，不要写成单次“全队生命池同时抬升”的专属捷径
- 大招的群体护盾必须复用统一 `Shield` 状态语义：对每个命中的友军分别附加护盾实例，而不是额外创建“团队共享护盾池”或“只要站在僧侣附近就持续免伤”的专属战斗状态
- 如果现有 `support_default` 更偏向后排站位，应优先补一个共享的“前排近战辅助站位偏好”或等价参数，而不是把僧侣写成只能靠专属移动脚本驱动的孤立英雄

## 设计边界

- 僧侣的核心职责是跟随前排稳住持续作战能力，不承担主坦克的接战、吃火力和强控职责
- 不应把僧侣做成兼具高额伤害、强开团控制和长期承伤的半战士核心
- 不应把僧侣做成能在远距离稳定救后排的全图保人辅助；它必须为群体治疗和群体护盾承担贴近阵线的站位成本
- 不应把大招扩成全图护盾、无敌或高额减伤类全能保命技能
- 不应在同一个英雄身上继续叠加分担伤害、嘲讽、击飞等过多坦克语义，避免它挤占坦克职业定位
- 第一阶段下，僧侣优先验证“近战瞬时普攻 + 贴身范围治疗小技能 + 贴身大范围群体护盾大招”这套副坦型辅助模板
