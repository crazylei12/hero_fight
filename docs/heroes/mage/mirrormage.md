# Mirrormage

## 基础信息

- 英文名：Mirrormage
- 中文名：镜法师
- 英雄 ID：`mage_006_mirrormage`
- 英雄分类：法师
- 标签：远程、分身、复制体、战术压制、镜像魔法
- 核心定位：中后排战术型法师，通过复制敌方关键英雄制造临时友方分身，把敌人的部分普攻与小技能能力短时间反转为己方压力
- 默认 AI 模板：`mage_default`
- 当前定位补充：作为 `mage_005_astromancer` 之后的第六个法师扩展原型，不替代 `Firemage` 的范围爆发、`Frostmage` 的减速控场、`Sandemperor` 的部署阵地、`Lightningmage` 的高频小控或 `Astromancer` 的延迟轰炸；主要验证“可被攻击的临时分身单位 + 复制安全规则 + 禁止套娃”的共享机制

## 当前数值倾向

- 镜法师本体的直接伤害应低于传统输出法师；他的主要价值来自复制体制造的临时人数、技能和目标压力
- 复制体强度必须明显弱于原单位，且存在时间较短，避免把镜法师变成稳定多带一名完整英雄
- 复制体的普攻和小技能强度主要通过自身攻击力、防御、生命等属性缩放自然降低；不再对 `clone-safe` 小技能的倍率、范围或控制时长做第二次折减
- 复制体可以被攻击、控制和击杀；敌方可以通过快速清掉复制体降低镜法师收益
- 复制敌方高威胁英雄时收益更高，但复制体不能继承大招、永久成长、召唤权限或分身权限，避免无限分身和跨系统套娃
- 镜法师缺少硬控、位移和爆发自保；被刺客贴脸、敌方快速击杀复制体或敌方阵容没有明显高价值复制目标时，应是他的主要弱点

## 当前实现基线

- 最大生命：`300`
- 攻击力：`32`
- 防御力：`9`
- 攻速：`0.862`，约等于 `1.16 秒 / 次`
- 移速：`3.7`
- 暴击率：`6%`
- 暴击伤害：`1.45`
- 攻击距离：`8.7`

## 普攻类型

- 类型：远程投射物普攻
- 释放目标：当前索敌目标；默认优先攻击最近的可攻击敌人，沿用现有法师和远程投射物英雄的共享索敌与受压后撤基线
- 效果：
  - 发射一枚镜光弹，对当前目标造成基于攻击力 `0.9` 倍率的单体伤害
  - 普攻只承担稳定补伤职责，不附带控制、减速、标记或额外召唤效果
  - 命中表现方向为小型镜片碎光，表现层只消费普攻命中事件，不参与伤害判定
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 普攻伤害倍率：基于攻击力 `0.9`
  - 攻击距离：`8.7`
  - 普攻间隔：`1.16 秒`
  - 投射物速度：`14.5`

## 小技能：Mirror Copy

- 技能类型：目标型临时分身单位创建
- 释放目标：优先选择施法距离内累计造成伤害最高的可复制敌方英雄；若没有统计结果，则回退到当前攻击目标，再回退到最近可复制敌方英雄
- 不可复制目标：
  - 已经是分身单位的目标
  - 部署型固定战斗代理体，例如沙兵、炮台、陷阱或阵地触发点
  - 普通召唤物、宠物、临时投射物、装饰单位
  - 死亡、复活等待、不可选中或明确标记为 `uncopyable` 的单位
  - 其他由共享分身系统判定为不适合作为模板的临时单位
- 效果：
  - 在目标敌方英雄附近的合法空位生成一个该英雄的镜像复制体
  - 复制体属于镜法师阵营，是镜法师的友方单位
  - 复制体可以自动移动、索敌、普攻，并尝试使用复制来源的小技能中被判定为 `clone-safe` 的部分
  - 复制体不具有大招，不会触发一局一次资源，不会复活，不参与己方或敌方固定出战人数统计
  - 复制体可以被敌方索敌、攻击、控制和击杀，死亡或持续时间结束后碎裂消失
  - 复制体死亡不计入击杀分，不触发普通英雄复活流程；但复制体造成的伤害、治疗、状态和击杀归属应能在日志中追溯到镜法师
  - 同一镜法师通过小技能同时最多保留 `1` 个镜像复制体；若未来冷却或特性导致重叠，新的小技能复制体默认替换旧的小技能复制体
- 复制体属性快照：
  - 复制体读取目标源单位的本局有效基础属性作为模板，包括英雄基础属性、战斗输入修正和选手修正后的运行时基线
  - 复制体不复制目标当前 Buff、Debuff、护盾、控制、持续伤害、持续治疗、临时形态、一次性大招状态或永久成长层数
  - 复制体生成时按缩放后的最大生命满血出现，不继承源目标当前血量比例
- 检测条件：
  - 小技能 CD 转好
  - 施法距离内存在至少 `1` 个可复制敌方英雄
  - 目标附近存在合法落点
  - 镜法师当前没有被硬控到无法施法
- 当前关键参数：
  - 施法距离：`10.2`
  - 冷却时间：`12 秒`
  - 复制体持续时间：`6.5 秒`
  - 生成偏移：目标身边约 `1.15` 世界单位，并通过共享合法落点分配避免重叠
  - 小技能复制体同时存在上限：`1`
  - 最大生命缩放：源单位本局有效最大生命的 `42%`
  - 攻击力缩放：源单位本局有效攻击力的 `42%`
  - 防御缩放：源单位本局有效防御的 `55%`
  - 攻速缩放：源单位本局有效攻速的 `85%`
  - 移速缩放：源单位本局有效移速的 `95%`
  - 普攻射程：复制源单位普攻射程
  - 小技能数值规则：使用源技能原始倍率与参数，不再额外降低伤害、治疗、护盾、状态幅度、范围半径或控制时长；基于攻击力的效果会因复制体自身攻击力缩放而自然变弱
  - 复制体初始小技能延迟：生成后至少等待 `1.15 秒` 才能首次尝试释放小技能

## 复制体小技能规则

- 复制体可以使用普通伤害、治疗、护盾、属性 Buff / Debuff、范围伤害、范围治疗、位移、减速、眩晕、击飞、嘲讽、集火标记等已经接入共享技能 / 状态系统的 `clone-safe` 小技能效果
- 复制体释放的小技能必须继续走统一目标选择、统一技能执行、统一数值结算、统一状态系统和统一日志入口
- 复制体释放 `clone-safe` 小技能时，技能倍率、范围、状态 `magnitude` 和控制时长默认沿用源技能配置；强度差异来自复制体自身属性缩放，而不是再对技能效果二次打折
- 若某个源技能存在不依赖施法者属性的强力固定值、特殊保护或特殊控制收益，应优先把该效果标记为非 `clone-safe`，而不是在镜法师这里为全部复制技能统一套低倍率
- 如果源英雄的小技能由多个效果组成，应允许共享系统只执行其中被标记为 `clone-safe` 的效果；不安全效果必须跳过并写入调试日志
- 复制体默认不能使用以下小技能能力：
  - 创建分身、复制体或其他 `CloneUnit`
  - 创建部署型固定战斗代理体、召唤物、宠物或陷阱
  - 复活、死亡防止、永久成长、局外成长、变身常驻形态
  - 一局一次类技能或依赖大招资源的效果
  - 需要特殊资源、特殊全局计数器或英雄私有运行时子系统的技能
  - 被动型小技能中的常驻监听、击杀参与成长或永久收益，除非以后专门标记为 `clone-safe`
- 若源英雄小技能完全不满足 `clone-safe` 条件，复制体仍可普攻，但不释放小技能

## 大招：Hall of Reflections

- 中文名：万象镜厅
- 技能类型：一局一次，多目标临时分身单位创建 / 镜像压制
- 释放目标：全图可复制敌方英雄群体，由大招决策半径搜索并筛掉分身、死亡、不可选中等非法模板
- 效果：
  - 镜法师展开短暂镜厅，从当前可复制敌方英雄中选择最多 `3` 个高威胁目标
  - 每个目标附近生成一个对应的镜厅复制体
  - 镜厅复制体属于镜法师阵营，可以自动移动、索敌、普攻，并尝试使用复制来源的小技能中被判定为 `clone-safe` 的部分
  - 镜厅复制体同样没有大招，不能复活，不能创建分身或召唤物，不计入击杀分
  - 镜厅复制体属性比小技能复制体更低，但数量更多，用于短时间制造战场人数和技能压力
  - 大招持续结束时，所有仍存活的镜厅复制体同时碎裂消失
- 目标选择优先级：
  - 累计造成伤害最高的敌方英雄
  - 当前攻击力较高的敌方输出或远程核心
  - 当前生命值较高、预期能稳定提供模板价值的敌方英雄
  - 最近可复制敌方英雄
- 检测条件：
  - 本局尚未释放过大招
  - 优先要求大招决策搜索半径 `56` 内至少存在 `3` 个可复制敌方英雄
  - 若开战 `28 秒` 后仍未满足，则阈值降低为至少 `2` 个可复制敌方英雄
  - 若开战 `44 秒` 后仍未满足，则阈值降低为至少 `1` 个可复制敌方英雄
  - 当前没有被硬控到无法施法
- 当前关键参数：
  - 技能自身施法距离：`0`；实际候选搜索由大招决策半径 `56` 控制
  - 最大复制数量：`3`
  - 镜厅复制体持续时间：`6 秒`
  - 生成偏移：各自目标身边约 `1.2` 世界单位，并通过共享合法落点分配避免重叠
  - 最大生命缩放：源单位本局有效最大生命的 `32%`
  - 攻击力缩放：源单位本局有效攻击力的 `32%`
  - 防御缩放：源单位本局有效防御的 `45%`
  - 攻速缩放：源单位本局有效攻速的 `80%`
  - 移速缩放：源单位本局有效移速的 `95%`
  - 小技能数值规则：使用源技能原始倍率与参数，不再额外降低伤害、治疗、护盾、状态幅度、范围半径或控制时长；基于攻击力的效果会因复制体自身攻击力缩放而自然变弱
  - 复制体初始小技能延迟：生成后至少等待 `1.35 秒` 才能首次尝试释放小技能
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`
  - 大招复制体与小技能复制体使用同一套共享分身规则；区别只在数量、持续时间和缩放参数

## 共享分身机制要求

- 镜法师不应拥有一套只服务自己的复制体底层；应沉淀为共享 `CloneUnit / 分身单位` 能力，后续其他英雄可以复用
- 分身单位不是 `部署型固定战斗代理体`：
  - 分身是可移动、可索敌、可普攻、可被选中、可受伤、可被控制的临时战斗单位
  - 部署型固定代理体默认不是英雄单位，通常不可移动、不可选中或不参与英雄索敌
- 分身单位需要至少记录：
  - `cloneId`
  - `cloneType`
  - `sourceHeroDefinition`
  - `sourceRuntimeUnitId`
  - `ownerUnitId`
  - `ownerTeamId`
  - `durationSeconds`
  - `statScale`
  - `skillParameterMode`
  - `canBasicAttack`
  - `canUseSmallSkill`
  - `canUseUltimate`
  - `canCreateClones`
  - `canBeTargeted`
  - `canTakeDamage`
  - `canBeControlled`
  - `copyBuffsMode`
  - `copySkillMode`
  - `deathMode`
- 镜法师复制体推荐配置：
  - `cloneType = MirrorCopy`
  - `canBasicAttack = true`
  - `canUseSmallSkill = true`
  - `canUseUltimate = false`
  - `canCreateClones = false`
  - `canBeTargeted = true`
  - `canTakeDamage = true`
  - `canBeControlled = true`
  - `copyBuffsMode = None`
  - `copySkillMode = BasicAttackAndCloneSafeSmallSkillOnly`
  - `skillParameterMode = UseSourceSkillParameters`
  - `deathMode = TimedDisappearOrKilled`

## 落地约束

- `Mirror Copy` 和 `Hall of Reflections` 都应优先复用共享 `CloneUnit` 能力表达，不应写成镜法师私有的临时英雄创建脚本
- 分身必须带有统一 `isClone` 或等价标记；所有复制目标选择、召唤目标选择和分身创建入口都必须默认排除 `isClone` 单位
- 分身不能释放 `CloneCreate`、`CreateDeployableProxy`、大招、一局一次技能、永久成长、复活、局外成长或其他明确禁止的效果
- 分身死亡不进入英雄复活流程，不增加击杀分，不占用 BP / 选手出战槽位，不回写任何英雄或选手静态资产
- 分身造成的伤害、治疗、护盾、状态和击杀应在事件与日志中同时保留“实际行动单位是分身”和“归属者是镜法师”两层信息
- 分身作为可攻击单位，应进入敌方索敌、范围命中、控制、受压后撤 threat、最小间距和位置收束等面向英雄单位的共享规则；这正是它与沙兵类固定代理体的区别
- 分身持续时间结束、被击杀、归属者死亡或战斗结束时都必须被统一清理，不能留下不可见可索敌单位、悬空状态或残留技能 CD
- 分身视觉表现可以先复用源英雄表现并叠加镜面 tint / 透明度 / 轮廓效果；表现层不应通过视觉颜色反推规则身份，逻辑身份必须来自运行时分身标记

## 设计边界

- 镜法师的核心职责是临时复制和战术压制，不承担稳定范围爆发主炮职责
- 不应把镜法师做成传统召唤海英雄；复制体数量必须有限，且持续时间短、强度缩放明显
- 不应让复制体继承大招、永久成长、复活、召唤权限或再次复制权限
- 不应让镜法师复制部署物、陷阱、沙兵、投射物、装饰单位或其他分身单位
- 不应把镜法师做成自带硬控、位移、无敌或高额自保的全能后排；他的生存仍依赖站位和队友保护
- 第一阶段下，镜法师优先验证“远程投射物普攻 + 目标型复制小技能 + 一局一次多目标镜像压制 + 共享分身单位规则”这套战术法师模板
