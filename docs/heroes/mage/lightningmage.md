# Lightningmage

## 基础信息

- 英文名：Lightningmage
- 中文名：雷电法师
- 英雄 ID：`mage_004_lightningmage`
- 英雄分类：法师
- 标签：远程、控制、多段伤害、持续压制
- 核心定位：中后排高频小控法师，通过所有伤害不断叠加 `Shock / 感电` 计数，在敌方前排顶进或突进英雄贴脸时频繁打出短眩晕；强度重心不是一次性爆发蒸发，而是让对手持续“打得很难受”
- 默认 AI 模板：`mage_default`
- 当前定位补充：作为 `mage_003_sandemperor` 之后的第四个法师扩展原型，不替代火法的范围爆发模板，也不替代冰法的减速控场模板，更不替代沙皇的阵地战模板；主要验证“主动直线路径技能 + 附带伤害事件监听被动 + 一局一次全场多段落雷”这条高频小控法师方向

## 当前数值倾向

- 单次伤害和单次爆发应明显低于 `Firemage`；雷电法师的主要收益来自连续命中后的短眩晕与稳定打断，而不是一键蒸发
- 普攻节奏应略快于传统法师，但不能快到仅靠普攻就稳定无限锁住同一目标；真正的控制频率应建立在“普攻 + 小技能 + 大招多段命中”的组合上
- 感电计数必须有明确持续时间并会因断档而掉空；如果雷电法师无法持续命中同一目标，控制压制力应明显下降
- 大招虽然覆盖全场，但 raw 伤害必须显著让位给“稳定给多人补感电、逼出团战短眩晕”这部分价值；默认不应把满血脆皮单独劈死

## 当前实现基线

- 最大生命：`290`
- 攻击力：`30`
- 防御力：`8`
- 攻速：`0.952`，约等于 `1.05 秒 / 次`
- 移速：`3.75`
- 暴击率：`10%`
- 暴击伤害：`1.7`
- 攻击距离：`8.9`

## 普攻类型

- 类型：远程投射物普攻
- 释放目标：当前索敌目标；默认优先攻击最近的可攻击敌人，继续沿用法师的中后排站位基线，不额外为雷电法师增加贴脸换血倾向
- 效果：
  - 发射一枚雷球，对命中的敌方目标造成一次基于攻击力 `0.75` 倍率的单体伤害
  - 普攻本体只承担“稳定补感电、维持压制节奏”的职责，不承担高单点爆发
  - 由于小技能附带的常驻被动监听，普攻命中也会为目标叠加 `Shock / 感电`
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 普攻伤害倍率：基于攻击力 `0.75`
  - 攻击距离：`8.9`
  - 普攻间隔：`1.05 秒`
  - 投射物速度：`15`

## 小技能：Thunderline

- 技能类型：固定方向路径伤害 + 附带伤害事件监听的共享计数被动
- 释放目标：优先锁定当前攻击目标所在方向；若当前攻击目标失效，则改为选择施法距离内敌方较密集区域中离自己最远的一名敌方英雄作为方向锚点，尽量让雷线穿过更多目标
- 效果：
  - 主动效果：朝锁定方向发射一列笔直雷线，对路径上的所有敌方英雄各造成一次基于攻击力 `1.35` 倍率的伤害
  - 同一目标在同一次释放中最多命中 `1` 次；雷线不在路径上反复多跳，也不生成持续留场区域
  - 被动效果：当敌方英雄受到来自雷电法师的任意伤害时，附加 `1` 层 `Shock / 感电`
  - 这套感电由雷电法师自己的普攻、小技能和大招共用同一组本地计数池
  - 感电持续 `4 秒`，重复触发时刷新持续时间，同一雷电法师本地共享池最大计数为 `3`
  - 当计数达到 `3` 时，目标立即受到一次 `Stun`
  - 眩晕持续 `0.6 秒`
  - 触发眩晕后，本次感电计数立即清零
  - 这条被动的核心收益是“频繁打断和卡节奏”，不是把雷电法师做成长时间硬控链角色
- 检测条件：
  - 小技能 CD 转好
  - 施法距离内存在合法方向锚点
  - 当前预计路径上至少可命中 `1` 名敌方英雄
- 当前关键参数：
  - 方向锚点施法距离：`9.5`
  - 路径长度：`11.2`
  - 路径宽度：`1.8`
  - 伤害倍率：基于攻击力 `1.35`
  - 冷却时间：`6 秒`
  - 感电持续时间：`4 秒`
  - 感电最大计数：`3`
  - 眩晕持续时间：`0.6 秒`

## 大招：Storm Verdict

- 技能类型：全场周期落雷伤害 / 团队级感电压制
- 释放目标：自己；每次脉冲都对所有当前存活的敌方英雄生效
- 效果：
  - 释放后进入 `5 秒` 的落雷期
  - 落雷期内每隔 `1 秒`，对所有当前存活的敌方英雄各劈下一道雷
  - 单次落雷造成基于攻击力 `1.10` 倍率的伤害
  - 每次落雷都视为“来自雷电法师的伤害”，因此也会继续为对应目标叠加 `Shock / 感电`
  - 若某个敌方英雄在大招期间死亡后又复活，则后续脉冲仍可重新命中该目标；每次脉冲都应按当下仍存活的合法敌方英雄重新收集目标
  - 大招核心收益是全队压血、批量补感电和稳定制造短眩晕窗口，不是单纯依靠 raw 伤害独立收割满血全队
- 检测条件：
  - 本局尚未释放过大招
  - 优先要求当前至少 `3` 名敌方英雄仍存活
  - 若当前至少 `2` 名敌方英雄已经带有 `2` 层感电，则显著提高释放优先级
  - 若开战 `40 秒` 后仍未等到理想窗口，则只要仍有至少 `2` 名敌方英雄存活即可释放
  - 若开战 `50 秒` 后仍未释放，则只要仍有至少 `1` 名合法敌方目标存活即可释放
- 当前关键参数：
  - 持续时间：`5 秒`
  - 脉冲次数：`5`
  - 脉冲间隔：`1 秒`
  - 单次落雷伤害：基于攻击力 `1.10`
  - 每跳目标范围：所有当前存活的敌方英雄
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 雷电法师当前定位为“第四个法师扩展原型”，不替代 `Firemage` 的范围爆发，也不替代 `Frostmage` 的减速压制，更不替代 `Sandemperor` 的部署阵地
- `Thunderline` 的主动部分应优先复用共享的固定方向路径伤害能力，不要单独再写一套“雷线专属命中脚本”
- 小技能附带的感电被动仍然只占用同一个正式小技能位；它应复用共享的“伤害事件驱动计数 + 阈值触发正式效果”能力，不额外新增独立被动栏
- `Shock / 感电` 应作为统一状态系统里的正式主题化计数标记处理；普攻、小技能和大招共用同一组本地叠层池，不要在 `RuntimeHero`、AI 或表现层偷偷维护一份雷电法师私有整数计数器
- 达到 `3` 层后触发的眩晕，必须直接复用正式 `Stun` 状态；清层也必须走统一状态系统和统一事件机制，不要写成“先手动改层数，再临时硬控目标”的双轨逻辑
- `Storm Verdict` 应优先复用共享的“周期脉冲 + 目标集合重收集”结构；每一跳都重新按“所有当前存活的敌方英雄”取目标，避免与死亡复活规则脱节
- 事件与日志至少应能明确记录：谁给谁叠加了感电、哪次伤害把感电叠到 `3` 层、哪次阈值触发导致了眩晕，以及大招每次脉冲命中了多少敌人

## 设计边界

- 雷电法师的核心职责是高频小控和节奏压制，不承担正面爆发主炮职责
- 不应把雷电法师做成“上来就直接群体长眩晕”的强开团法师；控制应明显依赖先叠层、后触发
- 不应把雷电法师做成高机动、自保极强或带无敌位移的法师；她应主要靠中后排站位和打断对手出手来保护自己
- 不应把 `Storm Verdict` 扩成“稳定单独劈死满血脆皮并顺便团灭全队”的全图核爆；它更像一次全队烦人窗口而不是全图秒杀按钮
- 不应把感电扩成永久不掉、无限留存的隐藏 debuff；一旦雷电法师失去持续命中，控制压力就应该明显回落
- 第一阶段下，雷电法师优先验证“远程投射物普攻 + 直线路径小技能 + 统一感电计数阈值眩晕 + 一局一次全场多段落雷”这套高频小控法师模板
