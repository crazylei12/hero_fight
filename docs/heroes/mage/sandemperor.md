# Sandemperor

## 基础信息

- 英文名：Sandemperor
- 中文名：沙皇
- 英雄 ID：`mage_003_sandemperor`
- 英雄分类：法师
- 标签：远程、阵地战、部署物、持续输出、区域压制
- 核心定位：中后排阵地法师，通过逐步在战场上布置沙兵建立固定火力区；只要敌人持续留在已有沙兵覆盖里，沙皇的持续压血会迅速变强；一旦被迫离开旧阵地，战斗力会明显回落，并需要花时间重新部署
- 默认 AI 模板：`mage_default`
- 当前定位补充：作为 `mage_002_frostmage` 之后的第三个法师扩展原型，不替代火法的范围爆发模板，也不替代冰法的减速控场模板，主要验证“固定部署物 + 普攻联动 + 一局一次全场重建阵地”这条阵地战法师方向

## 当前数值倾向

- 沙皇本体在没有沙兵参与时的直接输出应低于 `Firemage`，否则“脱离阵地后显著变弱”这条核心体验会不成立
- 当 `2 ~ 3` 个沙兵稳定覆盖同一交战区时，沙皇的持续压血能力应明显高于普通控场法师，但不应无条件超过真正的射手模板
- 沙皇不依赖暴击作为主要收益点，不应给到过高的暴击率或暴击伤害
- 沙皇缺少可靠位移和硬控自保；被迫频繁换边、被刺客反复切出原阵地，应该是他的主要弱点之一

## 普攻类型

- 类型：远程投射物普攻 + 沙兵联动突刺
- 释放目标：当前索敌目标；当前普攻默认优先攻击最近的可攻击敌人，沿用现有远程英雄的共享索敌基线，不额外为沙皇增加专属普攻索敌或站位特例
- 效果：
  - 沙皇发射一枚砂矛，对当前目标造成基于攻击力 `0.8` 倍率的单体伤害
  - 每个仍存活且其 `2.6` 范围内存在敌人的沙兵，会在沙皇完成本次普攻时同步突刺一次
  - 单个沙兵单次突刺造成基于攻击力 `0.55` 倍率的伤害，优先攻击沙皇当前普攻目标；若当前目标不在该沙兵附近，则改打该沙兵附近最近的敌人
  - 若当前交战区没有可参与的沙兵，沙皇只剩本体较弱普攻，体现脱离阵地后的明显掉强
- 检测条件：
  - 当前存在合法敌方目标
  - 目标进入攻击范围
  - 普攻间隔转好
- 当前关键参数：
  - 攻击距离：`8.8`
  - 普攻间隔：`1.10 秒`
  - 投射物速度：`14`
  - 本体伤害倍率：基于攻击力 `0.8`
  - 沙兵联动半径：`2.6`
  - 沙兵单次突刺伤害：基于攻击力 `0.55`

## 小技能：Raise Sandguard

- 技能类型：部署型固定战斗代理体
- 释放目标：在当前普攻目标周围部署；若当前普攻目标失效或超出施法距离，则本次小技能不应强行朝无关区域施放
- 效果：
  - 在目标点召唤一个沙兵
  - 沙兵不可移动，不作为英雄单位
  - 沙兵默认 `不可选中 + 无敌`，且不进入敌方普攻、单体技能或面向英雄的范围效果命中收集，不会被控制或 Debuff 当成正常英雄目标处理
  - 沙兵不会阻挡移动，也不会参与“被贴身”“受压后撤”“最小间距收束”这类面向英雄本体的行为规则
  - 沙兵自身不主动索敌，只在沙皇普通攻击时执行联动突刺
  - 沙兵持续时间结束后消失
  - 场上同时最多存在 `5` 个沙兵；超上限时，最早存在的沙兵立即消失
- 检测条件：
  - 小技能 CD 转好
  - 当前存在合法普攻目标
  - 当前普攻目标位于施法距离内，且其周围存在合法落点
  - 当前没有被硬控到无法施法
- 当前关键参数：
  - 施法距离：`9`
  - 冷却时间：`4 秒`
  - 沙兵持续时间：`12 秒`
  - 沙兵同时存在上限：`5`
  - 超上限替换规则：`移除最旧实例`

## 大招：Imperial Encirclement

- 技能类型：全场重建阵地 / 临时多点部署
- 释放目标：自己；对所有当前存活的敌方英雄生效
- 效果：
  - 立即在每个敌方英雄身边召唤一个临时沙兵
  - 每个临时沙兵在生成瞬间立刻对其对应敌方英雄执行一次突刺
  - 这些临时沙兵随后会继续沿用“沙皇普攻时联动突刺”的规则，持续一小段时间后消失
  - 临时沙兵同样不可移动、不作为英雄单位、不会阻挡移动，也不会让敌方英雄因为“被贴身”而进入额外逃离语义
  - 临时沙兵与小技能召唤出的沙兵共用同一套同时存在上限；若大招生成时超上限，则优先移除当前最旧的旧沙兵，相当于把沙皇的阵地瞬间重建到敌方当前站位附近
  - 大招的核心价值是“马上把旧阵地搬到当前战场中心”，而不是把沙兵做成会自动追人的独立召唤军队
- 检测条件：
  - 本局尚未释放过大招
  - 至少 `3` 名敌方英雄仍存活，且沙皇当前存在合法普攻目标
- 当前关键参数：
  - 生成时的单次突刺伤害：沿用沙兵基础突刺倍率
  - 临时沙兵持续时间：`5 秒`
  - 生成偏移：目标身边约 `1.2` 世界单位
  - 同时存在上限：`5`
- 当前关键约束：
  - 主动释放型大招本局仅能释放 `1 次`

## 落地约束

- 沙皇当前定位为“第三个法师扩展原型”，不替代 `Firemage` 的范围爆发，也不替代 `Frostmage` 的减速控场
- 沙兵应优先复用统一的“部署型固定战斗代理体”能力表达，不应直接做成独立 `RuntimeHero`、独立计分单位或独立 AI 单位
- 只要某次沙兵突刺是由沙皇普攻触发的，它的伤害、日志和击杀归属都应挂回沙皇的普攻链路，不要再额外发明一套“沙兵自己出手”的独立战斗身份
- 大招生成瞬间的那次突刺应归属 `Imperial Encirclement`；大招持续期间后续由沙皇普攻触发的联动突刺，仍优先归回普攻链路
- 沙兵不应进入默认敌方索敌集合，不应干扰远程单位的受压后撤 threat，也不应参与英雄最小间距和 body blocking；这些都应在共享规则中统一声明，不要散写在单英雄特例判断里
- 沙皇的英雄级 AI 当前应直接复用现有法师 / 远程英雄的共享索敌与站位基线，不额外引入“围绕自家部署物作战”的专属移动倾向
- 如果未来决定让沙兵可以被选中、被摧毁或主动索敌，必须先更新共享规划规则和本文档，不能在实现里悄悄放大成半独立召唤系统

## 设计边界

- 沙皇的核心职责是阵地构筑和留场压制，不承担高机动切后排法师职责
- 不应把沙皇做成依赖连续硬控链吃饭的控制核心
- 不应把沙皇做成“脱离沙兵也一样强”的普通远程输出位；没有阵地时必须明显变弱
- 不应把沙皇做成真正的召唤海量宠物英雄；沙兵是固定战场锚点，不是会自主追击和接管战场的独立军队
- 第一阶段下，沙皇优先验证“远程投射物普攻 + 固定部署沙兵 + 一局一次全场重建阵地”这套阵地战法师模板
